# PRD-0061: Dedicated FileWriteTool

## Overview
Introduce a dedicated `FileWriteTool` to provide a robust and safe mechanism for agents to create and modify files. This replaces the brittle and insecure practice of using shell redirections (`echo "..." > file.rb`) via `SafeShellTool`.

## Requirements

### 1. Functional
- **Actions**:
    - `write`: Write full content to a file (creates file if missing).
    - `append`: Append content to the end of a file.
    - `patch`: (Optional/Future) Apply a diff or substring replacement.
- **Parameters**:
    - `path`: String (relative to sandbox root).
    - `content`: String (raw file content).
    - `overwrite`: Boolean (default `true` for `write`).
- **Safety**:
    - Restricted to the sandbox directory (prevent `../` traversal).
    - Block modification of sensitive files (e.g., `.env`, `config/database.yml`) unless explicitly allowed in High tier.
    - Automatic directory creation (`mkdir -p`) for the target path.

### 2. Implementation Details
- New class `FileWriteTool < Agents::Tool`.
- Use `File.write` and `File.open(..., 'a')` for operations.
- Integrate with `AgentSandboxRunner` logic to ensure operations happen in the correct `cwd`.

### 3. Non-Functional
- **Reliability**: Handle encoding issues gracefully (default to UTF-8).
- **Security**: Validate that the resolved absolute path is within the sandbox `repo_dir`.

## Acceptance Criteria
- CWA can create a new controller file without using `SafeShellTool`.
- Attempting to write to a file outside the sandbox results in a `blocked` status.
- Writing to a nested path (e.g., `app/controllers/api/v1/test_controller.rb`) automatically creates the parent directories.
- `FileWriteTool` is registered and available in the `Agents::Registry`.
