# PRD-0060: Tiered Guardrails & Permissive CWA Mode

## Overview
Implement a tiered guardrail system for the multi-agent framework to allow for variable levels of autonomy and resource usage based on the agent's role and the execution environment. Specifically, enable a "High" tier for the Code Writing Agent (CWA) when running in a sandbox, increasing the `MAX_CALLS_PER_TURN` and relaxing shell/commit restrictions.

## Requirements

### 1. Functional
- **Tier Definition**:
    - Implement a `SecurityTier` concept (Low, Medium, High).
    - Map agents to tiers via configuration or context metadata.
- **Dynamic Limits**:
    - `MAX_CALLS_PER_TURN`:
        - Low: 5
        - Medium: 15
        - High: 30
    - `TIMEOUT_SECONDS`: Scale timeouts by 1.5x for High tier.
- **Permissive Shell (High Tier Only)**:
    - Expand `SafeShellTool` allowlist to include complex operations (`sh -c`, `grep -R`, `find`, `sed`, `awk`) ONLY when `SecurityTier == High` and `AGENT_SANDBOX_ACTIVE == 1`.
- **Soft Commit Gate (High Tier Only)**:
    - Allow `GitTool` to bypass the "tests must be green" check if `AI_COMMIT_SKIP_TESTS=true` is present in the environment or passed via tool arguments.
- **Tier Propagation**:
    - Ensure the `SecurityTier` is passed through the `AiWorkflowService` to the `Agents::Runner` and eventually to individual tools.

### 2. Implementation Details
- Update `Agents::Tool` base class (or individual tools) to respect a `security_tier` from `tool_context.context`.
- Update `AiWorkflowService` to assign tiers:
    - SAP Agent: Low
    - Coordinator Agent: Medium
    - CWA Agent: High (if sandbox initialized)
- Add `security_tier` to the `run.json` and `events.ndjson` for auditability.

### 3. Non-Functional
- **Security**: Tiers MUST NOT be upgradeable by the agent itself through tool calls.
- **Performance**: Negligible overhead for tier checking.
- **Observability**: Clearly log the active tier in every tool execution.

## Acceptance Criteria
- Tools correctly enforce different `MAX_CALLS_PER_TURN` based on the assigned tier.
- CWA in High tier can execute `sh -c` commands that are blocked in Low/Medium tiers.
- Git commits can be made in High tier even if the previous test run failed (if requested).
- Tier status is visible in `agent_logs`.
