# PRD-0062: RAG Service Refactor (Context Factory)

## Overview
Refactor the RAG (Retrieval-Augmented Generation) logic into a centralized `Ai::ContextFactory`. This service will serve as the single source of truth for providing context to agents, ensuring that SAP, Coordinator, and CWA all use consistent and up-to-date documentation and data snapshots.

## Requirements

### 1. Functional
- **Centralized Registry**: A single service that manages context assembly.
- **Context Types**:
    - `static`: Loads files defined in `context_map.md`.
    - `snapshot`: Loads user-specific financial data (anonymized).
    - `hybrid`: Merges static docs, snapshots, and repo-level metadata (e.g., file list, recent git log).
- **Agent-Specific Context**:
    - **SAP**: High priority on `eric_grok_static_rag.md` and routing logic.
    - **CWA**: High priority on codebase structure, testing conventions, and task-specific files.
- **Dynamic Selection**: Automatically select relevant documents based on the `query_type` (generate, debug, qa) or `correlation_id` history.

### 2. Implementation Details
- New service `app/services/ai/context_factory.rb`.
- Deprecate/Merge `SapAgent::RagProvider`.
- Update `AiWorkflowService` to call `Ai::ContextFactory.build(agent:, correlation_id:)`.
- Implement `ContextFactory#truncate` to honor token limits (default 4000 characters for now, but adjustable per model).

### 3. Non-Functional
- **Consistency**: Eliminate the risk of SAP seeing one version of a PRD while CWA sees another.
- **Performance**: Cached reads for static docs; efficient JSON parsing for snapshots.
- **Testability**: Unit tests for the factory to verify correct document merging and truncation.

## Acceptance Criteria
- All agents in a workflow receive context via the `ContextFactory`.
- The `eric_grok_static_rag.md` is correctly injected into SAP's prompt.
- CWA receives a "Codebase Snapshot" including a list of relevant files when starting a task.
- Token limits are enforced, preventing context overflow errors.
- `SapAgent::RagProvider` code is removed or fully redirected to the new factory.
