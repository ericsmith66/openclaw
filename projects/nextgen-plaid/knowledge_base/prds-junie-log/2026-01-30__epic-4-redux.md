## 8. Decisions & Rationale
- Decision: Use `llm_model` (not `model_name`) column on `PersonaConversation`.
  - Rationale: ActiveRecord reserves `model_name`; using it as a column name raises `ActiveRecord::DangerousAttributeError`.
- Decision: Make initial migration idempotent via `if_not_exists: true`.
  - Rationale: A failed migration attempt can leave partial schema changes; idempotency lets reruns succeed without requiring destructive DB resets.

## 9. Risks / Tradeoffs
- Title generation job currently uses `AgentHub::SmartProxyClient` directly (shared service) while the Epic mentions `SmartProxyClient` as shared.
  - Mitigation / follow-up: Consider extracting a shared `SmartProxyClient` facade later in Epic 4 if user chat + agent hub need a common interface.
