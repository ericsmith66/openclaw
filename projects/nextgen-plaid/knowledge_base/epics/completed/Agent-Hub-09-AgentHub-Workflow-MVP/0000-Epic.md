## Epic goal

Enable the **Agent Hub + Workflow system** so we can run the post-epic validation suite (the six stories in the vision doc §10/§11.3) through a full SDLC loop and get hands-on experience.

This epic delivers **platform capabilities + minimal UI surfaces**. It is not a persona work plan.

---

## Capabilities delivered (platform)

Source: `vision-agent_hub-workflow.md` §11.1.

1. **System health visibility** (proxy/workers/cable/cloudflare)
2. **Artifact store + SDLC phases** (structured + `jsonb` payload)
3. **Conversation ↔ workflow linking** (stable IDs, deep links)
4. **Status mover actions** (`/approve`, `/reject`, `/backlog`, routing)
5. **Minimal workflow execution surface outside Agent Hub** (workflow UI/background)

---

## UI surfaces

Source: `vision-agent_hub-workflow.md` §11.2.

- Agent Hub: minimal artifact link card + status mover actions
- Workflow UI (minimal): artifact list/detail + phase advancement
- Admin System Health (operator-facing): readiness checks
