# Epic 11: Consolidated SDLC Spike & Core Bridge

This epic consolidates the learnings and requirements from `Agent-Hub-09`, `Agent-Hub-10`, `vision-agent_hub-workflow.md`, and `gap_feedback.md` into a single, executable blueprint for the SDLC Bridge.

The **SDLC Bridge** is the infrastructure that connects human collaboration in the **Agent Hub** with autonomous execution in the **Workflow Engine**.

---

## Epic goal

Implement a **Human-Centric, Natural Language driven SDLC** with high readability and explicit verification gates. Shift from manual slash commands to a robust, intent-detecting Bridge architecture.

---

## Capabilities delivered (platform)

At the completion of this epic, the platform will support a fully integrated, natural-language-driven SDLC workflow with the following capabilities:

1.  **High-Fidelity Markdown Rendering:** All agent responses and artifact views support rich-text formatting (headers, lists, tables, code blocks), significantly improving readability and information density.
2.  **Autonomous Workflow Bridge:** A centralized `AgentHub::WorkflowBridge` service that autonomously detects agent intent from natural language responses (via structured tags), eliminating the need for manual slash commands.
3.  **Intent-Driven UI Interactions:** The UI dynamically generates descriptive action buttons (e.g., "Move to Analysis", "Finalize PRD") based on the bridge's intent detection. It supports both human-in-the-loop and silent background actions.
4.  **Context Integrity & Synchronization:** Guaranteed injection of the "Active Artifact" and automated "System Messages" upon phase changes to ensure the agent's memory remains synchronized with the database state.
5.  **Persistent Artifact Visibility:** A real-time synchronized "Artifact Preview" sidebar that serves as the single source of truth, showing the current phase and content of the active PRD or Technical Plan.
6.  **Developer Traceability:** A "Developer Mode" toggle allowing inspection of the exact RAG payload sent to the LLM, coupled with full audit trails linking artifact changes to specific RAG contexts.
7.  **Architectural Consolidation:** A clean, decoupled architecture where SDLC state transitions are managed by the Bridge rather than being scattered across communication channels (`AgentHubChannel`) and services.

---

## UI surfaces

- **Agent Hub Conversation:** Support for Markdown and automated confirmation bubbles.
- **Artifact Preview Sidebar:** Persistent view of the active artifact, phase, and content.
- **Confirmation Bubbles:** Enhanced with descriptive, context-aware labels.

## Combined Human Test Criteria (Full SDLC)

This section outlines how to verify the combined capabilities of the epic through a complete SDLC flow.

| Phase | Human Action | Expected Result | PRDs Exercised |
| :--- | :--- | :--- | :--- |
| **Discovery** | Human: "SAP, I need a new billing module." | Agent responds with a Markdown-formatted draft. `[ACTION]` tag detected. | 11A, 11B |
| **Analysis** | Human clicks "Move to Analysis". | Sidebar updates to "Analysis". Conversation context now includes the Billing PRD. | 11C, 11D, 11E |
| **Planning** | Human: "Generate a technical plan." | Agent (using PRD context) generates a plan. "Start Planning" button appears. | 11B, 11C, 11D |
| **Execution** | Human clicks "Start Implementation". | Sidebar shows "Implementation". CWA starts coding with full PRD + Plan context. | 11C, 11D, 11E |
| **Verification** | Human checks logs and architecture. | No slash commands used. All state changes mediated by `WorkflowBridge`. | 11F |
