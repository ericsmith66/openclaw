# Epic 4: Workflow Persistence & Management (Agent-Hub-04)

### Epic Goal
Formalize the DB schema for AI runs (conversations), ensuring history is durable and states (pending/approved) are explicit—foundational for long-running autonomous flows.

### Scope
AiWorkflowRun model with metadata JSONB; status machine (draft, pending, approved, failed); association with User/Owner; audit logs for approval.

### Non-Goals
Real-time streaming (Epic 2 covers); smart commands (Epic 5).

### Dependencies
Epic 3 for sidebar listing; existing PG setup.

### Risks/Mitigations
JSONB bloat → define schema/validator; orphaned runs → soft-delete + cleanup worker.

### End-of-Epic Capabilities
- Conversations persist after reload.
- Users approve "Pending" runs via UI button.
- Metadata stores model parameters used (temp, top-p).
- Filter sidebar by "Approved" vs "Awaiting".

### Safety Policy (Canonical)
- RLS ensures owner-only data access.
- Approval required for high-risk actions.
- Logs include approval timestamp.

### Model Discovery
Store discoverable model ID in metadata during run creation.
