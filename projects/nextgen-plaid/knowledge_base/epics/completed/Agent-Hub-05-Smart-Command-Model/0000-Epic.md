# Epic 5: Smart Command & Model Engine (Agent-Hub-05)

### Epic Goal
Integrate the SmartProxy/Ollama engine into Agent Hub, allowing users to trigger commands (/handoff, /search) and see real-time agent "thinking" via the Cable pane—completing the interactive hub.

### Scope
Command parser (regex/service); SmartProxy integration (v1/chat); real-time broadcast of thought-tokens; "Model Discovery" implementation (fetching local models).

### Non-Goals
Persistence logic (Epic 4 covers); heavy RAG implementation (future).

### Dependencies
Epic 2 (Streaming), Epic 3 (Input), Epic 4 (Persistence).

### Risks/Mitigations
Model latency → "thinking" indicator + timeout; invalid commands → help hint fallback.

### End-of-Epic Capabilities
- Users type `/handoff` and see conductor transfer.
- Live streaming of agent "thoughts" (hidden in final UI, visible in Hub).
- Model selector reflects available local Ollama models.
- Commands log to audit trail.

### Safety Policy (Canonical)
- Only allow safe, non-destructive commands.
- Local model execution; no cloud data leak.
- Timeout prevents hung runs.

### Model Discovery
Dynamic fetch from `SmartProxy /v1/models` to populate UI dropdown.
