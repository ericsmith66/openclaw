# Epic 2: Real-time Streaming Core (Agent-Hub-02)

### Epic Goal
Implement Action Cable for dynamic pane updates, enabling "alive" monitoring of agent activity—critical for trust in autonomous flows like escalations, without overloading channels.

### Scope
Dedicated channel with stream caps; token streaming, typing indicators, model badges on bubbles; polling fallback.

### Non-Goals
Full chat interactions (input/commands); persistence ties.

### Dependencies
Epic 1 for layout/pane; Action Cable config.

### Risks/Mitigations
Channel overload → cap at 5 streams + tests; disconnects → polling fallback.

### End-of-Epic Capabilities
- Real-time token appends to pane with gray thoughts.
- Pulsing "…" on processing.
- Model badges visible on responses.
- Fallback ensures degraded reliability.

### Safety Policy (Canonical)
- Broadcasts local-only; no external data.
- Cap prevents DoS-like overload.
- Logs events without PII.

### Model Discovery
Query SmartProxy /v1/models for badge prep; fallback ENV.
