# Epic 7: Persistence Wiring & Lifecycle (Agent-Hub-07)

### Epic Goal
Move the Agent Hub from stubbed data to a fully persistent system using `AiWorkflowRun`, ensuring a robust lifecycle for conversations including auto-titling, status badges, soft-deletes, and external backlog integration.

### Scope
Wiring Sidebar to `AiWorkflowRun` (real data); Auto-titling logic; "Human attention" status badges; Soft-delete/Archive lifecycle; DB-backed external backlog (JSONB); `AiWorkflowRun` enhancements (`name`, `description`); Cleanup worker; Redis state for Action Cable.

### Non-Goals
UI Component creation (Epic 6); Multi-agent specific monitoring (Epic 8).

### Dependencies
Epic 4 (Workflow Persistence), Epic 3 (Sidebar).

### Risks/Mitigations
Data growth → Soft-delete policy + Cleanup worker; Performance → JSONB indexing.

### End-of-Epic Capabilities
- Sidebar reflects real database re
- cords for the owner.
- Conversations are auto-titled based on content.
- Pending approvals are flagged with red badges.
- Deleted conversations are archived (soft-delete).
- Approved artifacts can be sent to an external JSONB backlog.
- Persona state persists across reloads.

### Safety Policy
- Soft-delete only (archiving).
- RLS on all run access.
- Audit logging for all status transitions.
