# PRD-AH-OLLAMA-TOOL-00: Conversation History Normalization

Epic: EPIC-AH-OLLAMA-TOOL-USE
Status: Ready for Implementation
Priority: Critical (Prerequisite for all other PRDs)

---

## Overview

Prerequisite fix for spike failure. Ensure Ollama receives tool call arguments
as JSON objects (not strings) to prevent HTTP 400 unmarshal errors in mixed-
provider histories.

---

## Problem Statement

During SDLC E2E work with mixed providers (Grok -> Ollama), we encountered:

ERROR: HTTP 400 from Ollama API
MESSAGE: "cannot unmarshal string into tool_calls.function.arguments"

ROOT CAUSE:
- Grok/OpenAI-style assistant messages include tool_calls[].function.arguments as JSON STRING
- Ollama /api/chat expects tool_calls[].function.arguments as JSON OBJECT
- Without normalization, Ollama rejects the entire request

IMPACT:
- Blocks mixed-provider workflows
- Prevents Ollama from consuming histories generated by other providers
- Makes tool execution unreliable in agent workflows

---

## Requirements

### Functional Requirements

1. Walk every message in payload['messages'] array
2. For each message with tool_calls array:
   - Check each tool_call.function.arguments field
   - If String: JSON.parse it to Hash
   - If parse succeeds: replace with Hash object
   - If parse fails: replace with {} and log error
   - If already Hash: leave unchanged
3. Return normalized payload (deep copy to avoid mutation)
4. Log count of normalized tool calls at DEBUG level

### Non-Functional Requirements

- Performance: O(n) on message count; negligible overhead for <50 messages
- Safety: Deep copy to prevent mutation of original payload
- Logging: Follow knowledge_base/prds/prds-junie-log/junie-log-requirement.md format
- Resilience: Graceful handling of malformed JSON (replace with {} instead of failing)

## Implementation

### Location

FILE: smart_proxy/lib/ollama_client.rb
METHOD: normalize_tool_arguments_in_payload (private)
CALLED FROM: chat_completions method (STEP 1)

### Code Reference

```ruby
def normalize_tool_arguments_in_payload(payload)
  normalized = payload.dup
  return normalized unless normalized['messages'].is_a?(Array)

  normalized_count = 0

  normalized['messages'] = normalized['messages'].map do |msg|
    next msg unless msg['tool_calls'].is_a?(Array)

    msg_copy = msg.dup
    msg_copy['tool_calls'] = msg['tool_calls'].map do |tc|
      tc_copy = tc.dup
      args = tc.dig('function', 'arguments')

      if args.is_a?(String)
        tc_copy['function'] = tc['function'].dup
        tc_copy['function']['arguments'] = JSON.parse(args) rescue {}
        normalized_count += 1
      end

      tc_copy
    end

    msg_copy
  end

  if normalized_count > 0
    log_debug(
      event: 'tool_arguments_normalized',
      count: normalized_count,
      provider: 'ollama'
    )
  end

  normalized
end
```

## Test Cases

### Unit Tests (spec/ollama_client_spec.rb)

```ruby
describe '#normalize_tool_arguments_in_payload' do
  it 'converts string arguments to hash' do
    payload = {
      'messages' => [
        {
          'role' => 'assistant',
          'tool_calls' => [
            {
              'function' => {
                'name' => 'search',
                'arguments' => '{"query": "test"}'
              }
            }
          ]
        }
      ]
    }

    result = client.send(:normalize_tool_arguments_in_payload, payload)
    args = result['messages'][0]['tool_calls'][0]['function']['arguments']

    expect(args).to be_a(Hash)
    expect(args['query']).to eq('test')
  end

  it 'handles bad JSON gracefully' do
    payload = {
      'messages' => [
        {
          'role' => 'assistant',
          'tool_calls' => [
            {
              'function' => {
                'name' => 'search',
                'arguments' => 'invalid json'
              }
            }
          ]
        }
      ]
    }

    result = client.send(:normalize_tool_arguments_in_payload, payload)
    args = result['messages'][0]['tool_calls'][0]['function']['arguments']

    expect(args).to eq({})
  end

  it 'preserves hash arguments unchanged' do
    payload = {
      'messages' => [
        {
          'role' => 'assistant',
          'tool_calls' => [
            {
              'function' => {
                'name' => 'search',
                'arguments' => { 'query' => 'test' }
              }
            }
          ]
        }
      ]
    }

    result = client.send(:normalize_tool_arguments_in_payload, payload)
    args = result['messages'][0]['tool_calls'][0]['function']['arguments']

    expect(args).to be_a(Hash)
    expect(args['query']).to eq('test')
  end

  it 'handles messages without tool_calls' do
    payload = {
      'messages' => [
        { 'role' => 'user', 'content' => 'Hello' }
      ]
    }

    result = client.send(:normalize_tool_arguments_in_payload, payload)

    expect(result['messages'][0]['role']).to eq('user')
    expect(result['messages'][0]['content']).to eq('Hello')
  end

  it 'logs normalization count' do
    expect(client).to receive(:log_debug).with(
      event: 'tool_arguments_normalized',
      count: 1,
      provider: 'ollama'
    )

    payload = {
      'messages' => [
        {
          'role' => 'assistant',
          'tool_calls' => [
            {
              'function' => {
                'name' => 'search',
                'arguments' => '{"query": "test"}'

    client.send(:normalize_tool_arguments_in_payload, payload)
  end
end
```

```ruby
def test_ollama_tool_calling_with_normalization
  res = http_post_json("/v1/chat/completions", {
    model: ENV.fetch("OLLAMA_MODEL", "llama3.1:70b"),
    messages: [
      {
        role: "assistant",
        content: nil,
        tool_calls: [
          {
            id: "call_abc123",
            type: "function",
            function: {
              name: "search",
              arguments: '{"query": "test"}'  # STRING (Grok style)
            }
          }
        ]
      },
      {
        role: "tool",
        tool_call_id: "call_abc123",
        content: "Search result"
      },
      {
        role: "user",
        content: "Summarize that result"
      }
    ],
    stream: false
  })

  assert_equal 200, res.code.to_i
  body = JSON.parse(res.body)
  assert body.dig("choices", 0, "message", "content").present?
end
```
