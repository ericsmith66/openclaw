# Epic 1: Scaffold & Layout Foundation (Agent-Hub-01)

### Epic Goal
Establish the secure, isolated base for Agent Hub at /agent_hub, providing owner-only access with a minimal layout—foundational for all subsequent real-time and interactive features, ensuring no interference with Plaid syncs and aligning with Vision 2026 privacy.

### Scope
Routing, authentication, Turbo Frame structure, header/footer globals, and mobile spike; reuse Devise/Pundit for auth; DaisyUI/Tailwind for styling; no DB interactions yet.

### Non-Goals
Real-time elements (Action Cable); UI components like tabs/sidebar; full responsive testing beyond spike.

### Dependencies
Existing auth gems (Devise, Pundit); Rails 8 setup; no prior epics.

### Risks/Mitigations
Auth bypass → enforce require_owner filter + tests; layout bloat → keep stripped, <500ms load via benchmarks.

### End-of-Epic Capabilities
- Secure /agent_hub route loads for owners only.
- Basic Turbo Frames enable future SPA-like behavior.
- Header link to Mission Control and footer disclaimer persist state.
- Mobile spike validates usability on small screens.

### Safety Policy (Canonical)
- Owner-only via RLS/filter; no data exposure.
- No network calls; local-only.
- Deny non-owners with 403.

### Model Discovery
Not applicable (no AI yet); fallback to ENV for future.
