# PRD-3-16: Snapshot Export Button

## Overview
Add a dropdown export button on the dashboard for downloading latest snapshot as JSON or CSV, leveraging PRD-2-08 export API endpoint.

## Status
**Branch**: `feature/prd-3-16-export-button`
**Dependencies**: PRD-3-15 (Transactions Summary) ⬜

## Log Requirements
Junie: Read `<project root>/knowledge_base/prds/prds-junie-log/junie-log-requirement.md` and log your plan/questions.

## Requirements

### Functional
- DaisyUI dropdown: "Export Snapshot" → options "JSON" and "CSV"
- Triggers download via existing API route from PRD-2-08 (e.g., GET `/api/snapshots/:id/download?format=json`)
- Filename: `networth-snapshot-YYYY-MM-DD.json` or `.csv`
- **JSON**: Exports full `snapshot.data`
- **CSV**: Uses schema from `financial_snapshot_data_schema.md`: `Account,Symbol,Name,Value,Percentage`
- Consider ZIP archive if exporting multiple CSVs (holdings, transactions, accounts)

### Non-Functional
- Secure (user-scoped only via RLS)
- Fast generation (<2s)
- Touch targets ≥44×44px

### Rails-Specific
- Use `link_to` or `button_to` with `data: { turbo: false }` for download
- Controller action from PRD-2-08 handles `send_data`
- Place in dashboard header or card footer

### Error Scenarios & Fallbacks
- Empty snapshot → disabled button with tooltip "No data to export"
- Export failure → flash error "Export failed, please try again"
- Network error → show retry option

## Architectural Context
Button in layout or header partial. RLS ensures only own snapshot exported. No new endpoint needed (uses PRD-2-08 API).

## Acceptance Criteria
- Dropdown appears and opens correctly
- JSON option downloads valid JSON matching latest `snapshot.data`
- CSV option downloads tabular file with correct headers and data
- Filename includes date (YYYY-MM-DD format)
- Empty snapshot → graceful message or disabled button
- Download initiates without page reload
- Touch targets ≥44×44px

## Test Cases
- Integration: Click options → assert download response headers/content-type
- Manual: Verify file contents match Rails console snapshot
- ViewComponent Preview: enabled, disabled_empty_snapshot
