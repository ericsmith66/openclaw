# PRD-3-17: Refresh Snapshot / Sync Status Widget

## Overview
Implement a refresh button with sync status badge (e.g., "Up to date", "Syncing...", last sync time) and rate limiting (1/min via rack-attack), using Turbo Streams for async feedback. Triggers `FinancialSnapshotJob`.

## Requirements

### Functional
- Button enqueues `FinancialSnapshotJob` if not rate-limited (1/min via rack-attack)
- Badge updates via Turbo Stream broadcast:
  - Pending: spinner + "Syncing... usually takes 30-90 seconds"
  - Success: "Up to date" + timestamp
  - Error: error badge + failure reason
- Show last sync timestamp
- On 429 rate limit → alert "Refresh limit reached — try again in [countdown]" with JS countdown timer

### Non-Functional
- Prevent API abuse via rack-attack
- Visible feedback at all stages
- Touch targets ≥44×44px
- Responsive header placement

### Rails-Specific
- **rack-attack configuration**:
  ```ruby
  Rack::Attack.throttle("snapshot_sync/user", limit: 1, period: 60) do |req|
    req.user&.id if req.path.match?(%r{/net_worth/sync})
  end
  ```
- Turbo Frame `#sync-status-frame`
- Job broadcasts via:
  ```ruby
  user.broadcast_replace_to(
    "net_worth:sync_status:#{user.id}",
    target: "sync-status",
    partial: "net_worth/sync_status",
    locals: { status: :complete, snapshot: @snapshot }
  )
  ```
- Countdown timer: Stimulus controller or vanilla JS

### Error Scenarios & Fallbacks
- Job failure → error badge + broadcast failure reason
- Websocket disconnect → fallback to page refresh prompt
- Network error → flash alert + retry link
- Rate limit hit → show countdown "Try again in 45 seconds"

### Security
- **Critical**: Verify Turbo Stream channels user-scoped
- Add test: assert no cross-user broadcast leak
- Channel authentication in `app/channels/application_cable/connection.rb`:
  ```ruby
  identified_by :current_user
  def connect
    self.current_user = env['warden'].user || reject_unauthorized_connection
  end
  ```

## Architectural Context
Widget component in Turbo Frame. Controller action `/net_worth/sync` enqueues job and responds with Turbo Stream. `FinancialSnapshotJob#perform` broadcasts on complete/failure. Rate limit enforced at rack-attack layer.

## Implementation Notes (as built)
- Route: `POST /net_worth/sync` → `NetWorth::SyncsController#create`
- UI: `NetWorth::SyncStatusWidgetComponent` + `net_worth/_sync_status` partial
- Turbo stream name (user-scoped): `net_worth:sync_status:<user_id>`
- Rate limit: `rack-attack` throttle `snapshot_sync/user` (1/min)
- 429 response: Turbo Stream replace of `#sync-status` with countdown UI
- Job broadcasts: `FinancialSnapshotJob` broadcasts `:complete`/`:error` states to the user-scoped stream
