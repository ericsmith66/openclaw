# PRD-3-13: Performance View

## Overview
Render 30-day net worth performance as interactive Chartkick line chart using pre-computed historical data from latest snapshot JSON. Refines performance subdir and PRD-2-06 trends data.

## Status
**Branch**: `feature/prd-3-13-performance-view`
**Dependencies**: PRD-3-12 (Sector Weights) ⬜

## Requirements

### Functional
- Line chart plotting daily total NW from `data['historical_totals']` array
- Tooltips with date, value, day-over-day delta
- **CRITICAL**: Use ONLY `historical_totals` from latest snapshot—do NOT query multiple snapshot rows

### Non-Functional
- No additional DB queries beyond latest snapshot fetch
- Responsive scaling
- Touch targets ≥44×44px

### Rails-Specific
- `app/components/net_worth/performance_component.rb` in Turbo Frame `#performance-chart-frame`
- **Add inline comment**: `# Performance: using pre-rolled historical_totals from latest snapshot only`
- Include `<table class="sr-only">` fallback for accessibility
- Chartkick line_chart with date formatting

### Error Scenarios & Fallbacks
- Fewer than 2 data points → "Insufficient history for trend. Check back after a few days."
- Nil data → `EmptyStateComponent(context: :data_missing)`
- Corrupt JSON → log to Sentry + "Data temporarily unavailable"

## Architectural Context
```ruby
# Controller
historical_data = data['historical_totals'] || []
# Array: {date: String, total: Float, delta: Float}
# Example: [{date: "2026-01-20", total: 5_000_000, delta: 12_345}, ...]
```

Chartkick handles rendering. No DB queries. RLS filters to user.

## Acceptance Criteria
- Line accurately plots historical totals over 30 days
- Tooltips display date, NW value, and delta
- Fewer than 2 points → insufficient history message
- Responsive and zoom-friendly
- Accessible: table fallback with same data
