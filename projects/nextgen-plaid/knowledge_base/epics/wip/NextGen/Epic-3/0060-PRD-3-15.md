# PRD-3-15: Transactions Summary View

## Overview
Create transactions summary cards displaying current-month income, expenses, and net flow. Pulls from snapshot's transaction aggregates; enhances income/transactions subdirs from PRD-2-09. **Detail link deferred to Epic 4**.

## Requirements

### Functional
- Three DaisyUI stat cards: Income (+green with ↑), Expenses (-red with ↓), Net (color by sign)
- Values formatted as currency
- **No detail link** (deferred to Epic 4) OR placeholder link `#` with tooltip "Full transactions view coming soon"
- Cards show last 30 days of data

### Non-Functional
- Responsive grid layout (3-column desktop, stacked mobile)
- Fast static render from JSON
- Touch targets ≥44×44px

### Rails-Specific
- `app/components/net_worth/transactions_summary_component.rb` + ERB
- Assume `data['transactions_summary']`: `{month: {income: Float, expenses: Float, net: Float}}`
- DaisyUI stat card component
- Use symbols ↑/↓ with colors for accessibility

### Error Scenarios & Fallbacks
- No data → "No recent transactions—sync accounts"
- Corrupt JSON → log to Sentry + "Data temporarily unavailable"

## Architectural Context
```ruby
summary = data['transactions_summary'] || {}
# Hash: {month: {income: Float, expenses: Float, net: Float}}
```

Controller provides summary hash from snapshot JSON. Component focuses on presentation.

## Acceptance Criteria
- Cards show accurate monthly totals from latest snapshot
- Colors applied correctly (positive green with ↑, negative red with ↓)
- Net card reflects income - expenses
- **No functional detail link** (placeholder only or omitted)
- No data → empty state with sync CTA
- Responsive: 3-column desktop, stacked mobile
- ARIA labels for stats
