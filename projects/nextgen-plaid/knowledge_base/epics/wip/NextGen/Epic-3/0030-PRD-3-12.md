# PRD-3-12: Sector Weights View

## Overview
Implement sector exposure view with bar chart and sortable table displaying % and $ values per sector. Sources `sector_weights` JSON; builds on sectors subdir from Epic 2.

## Status
**Branch**: `feature/prd-3-12-sector-weights`
**Dependencies**: PRD-3-11 (Allocation View) ⬜

## Requirements

### Functional
- Chartkick horizontal bar chart for top sectors
- DaisyUI table below with sortable columns (sector name, %, value)
- Hover tooltips on bars: "Technology: 28% ($2.3M)"
- Use sector colors from schema color palette
- Sort via Stimulus if <10 sectors (client-side), else Turbo with params

### Non-Functional
- Responsive: table collapses or horizontal scroll on mobile
- Touch targets ≥44×44px

### Rails-Specific
- `app/components/net_worth/sector_weights_component.rb` + ERB
- Turbo Frame `#sector-table-frame`
- Chartkick + DaisyUI table
- Skeleton loader while loading

### Error Scenarios & Fallbacks
- No equity holdings → "No sector data available (equity holdings only)"
- Corrupt JSON → log to Sentry + "Data temporarily unavailable"
- Chart error → fallback table only

## Architectural Context
```ruby
sector_data = data['sector_weights'] || []

```

Component handles presentation; secure via RLS.

## Acceptance Criteria
- Bar chart and table accurately reflect JSON data
- Table sorting updates view dynamically
- Tooltips show on bar hover (tap on mobile)
- Mobile: table stacks or scrolls cleanly
- Empty state message shown when no data
- Accessible: proper table headers/labels, ARIA sort indicators
