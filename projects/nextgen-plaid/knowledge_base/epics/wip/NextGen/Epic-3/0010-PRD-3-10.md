# PRD-3-10: Net Worth Summary Card Component

## Overview
Implement a prominent hero summary card for total net worth with daily and 30-day deltas ($ and %) including color coding, explanatory tooltips, and last updated timestamp. Replaces any basic NW display in the PRD-2-09 dashboard layout, giving users instant wealth snapshot and change awareness.

## Requirements

### Functional
- Display formatted total net worth with `number_to_currency`
- Day delta ($ and %) with proper sign: +$45,200 / +3.8%
- 30-day delta ($ and %) with proper sign: -$12,500 / -1.2%
- Color coding: Positive = green with ↑ symbol, negative = red with ↓ symbol
- Hover tooltips (tap on mobile):
  - Day: "Day change vs. most recent prior snapshot"
  - 30-day: "30-day change from earliest in window"
- **"Last updated [time_ago]" timestamp** showing `snapshot.created_at` ("3 hours ago")
- Empty state: use `NetWorth::EmptyStateComponent(context: :no_snapshot)`

### Non-Functional
- Render <500ms
- Fully responsive (stack vertically on mobile)
- Touch targets ≥44×44px
- ARIA labels for deltas and tooltips
- No client-side computation (all from snapshot JSON)

### Rails-Specific
- Create ViewComponent `app/components/net_worth/summary_card_component.rb` inheriting from `NetWorth::BaseCardComponent`
- Matching ERB template in Turbo Frame `#net-worth-summary-frame`
- Integrate into main dashboard view `app/views/net_worth/index.html.erb`
- Use Rails helpers: `number_to_currency`, `number_to_percentage`, `time_ago_in_words`
- DaisyUI stat/card classes with skeleton loader

### Error Scenarios & Fallbacks
- Nil snapshot → `EmptyStateComponent(context: :no_snapshot)` with CTA to sync
- Corrupt JSON → log to Sentry (`epic:3, prd:"3-10", component:"SummaryCard"`) + show "Data temporarily unavailable"
- Turbo failure → flash alert + refresh link
- Missing data keys → defensive `data['key'] || {}` fallbacks

## Architectural Context
**MVC**: `NetWorth::DashboardController#show` fetches:
```ruby
snapshot = FinancialSnapshot.latest_for_user(current_user)
data = snapshot&.data || {}
```

Component receives:
```ruby
summary: data['net_worth_summary'] || {
  total: 0,
  day_delta_usd: 0,
  day_delta_pct: 0.0,
  thirty_day_delta_usd: 0,
  thirty_day_delta_pct: 0.0
}
timestamp: snapshot&.created_at
```

**Security**: Application-level scoping via `current_user.financial_snapshots`. Devise authentication required.

## Acceptance Criteria
- Total net worth shows correctly formatted (e.g., $12,345,678.90)
- Deltas display with proper sign, color (green/red), symbol (↑/↓), and formatting
- Tooltips appear on hover (tap on mobile) with accurate explanations
- Timestamp shows relative time ("3 hours ago")
- No snapshot → shows empty state message with sync CTA
- Card responsive: full-width desktop, stacked mobile
- ARIA labels ensure screen reader compatibility
- No unescaped JSON or sensitive leaks in HTML
- Touch targets meet 44×44px minimum

## Test Cases

### Unit Tests (RSpec)
```ruby
describe NetWorth::SummaryCardComponent do
  it "renders formatted net worth" do
    summary = { total: 1_234_567.89, day_delta_usd: 12_345, day_delta_pct: 1.2, thirty_day_delta_usd: -45_678, thirty_day_delta_pct: -3.8 }
    component = render_inline(described_class.new(summary: summary, timestamp: 3.hours.ago))

    expect(component.css('.net-worth-total').text).to include("$1,234,567.89")
    expect(component.css('.delta-day.positive').text).to include("+$12,345")
    expect(component.css('.delta-30d.negative').text).to include("-$45,678")
  end

  it "handles empty state" do
    component = render_inline(described_class.new(summary: nil, timestamp: nil))
    expect(component).to have_css('.empty-state')
  end
end
```

### Integration Tests
```ruby
test "dashboard displays summary card with correct values" do
```
