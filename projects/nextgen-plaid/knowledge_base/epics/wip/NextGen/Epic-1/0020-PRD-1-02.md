### PRD-1-02: Account Strategy & Trust Association Extensions

**Overview**
Add `asset_strategy:string` and `ownership_lookup_id:integer` (nullable foreign key to OwnershipLookup) to Account model.

**Requirements**
- **Functional**:
  - Migration: Add `asset_strategy:string` and `ownership_lookup_id:integer` to accounts table
    - `asset_strategy` defaults to `"unknown"`
  - Add `belongs_to :ownership_lookup, optional: true` association
  - Add index on `ownership_lookup_id` for fast lookups
  - Add DB foreign key constraint from `accounts.ownership_lookup_id` to `ownership_lookups.id`
    - Delete behavior: restrict deletion of `ownership_lookups` when referenced by `accounts`
- **Non-Functional**:
  - Nullable fields (legacy accounts unaffected)

**Acceptance Criteria**
- Migration runs cleanly
- Account can be associated with an OwnershipLookup
- Queries on ownership_lookup_id are performant

**Workflow**
- Branch: `feature/prd-1-02-account-extensions`
- Command: `rails g migration AddStrategyAndOwnershipLookupToAccounts asset_strategy:string ownership_lookup_id:integer`
