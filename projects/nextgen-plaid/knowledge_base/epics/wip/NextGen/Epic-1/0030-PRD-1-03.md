### PRD-1-03: Ownership Lookup Model & Restricted CRUD

**Overview**
Extend `OwnershipLookup` (introduced in PRD-1-02) with ownership classification metadata; restrict CRUD to admin/parent roles using Pundit.

**Requirements**
- **Functional**:
  - Ownership lookup model: `name:string`, `ownership_type:string`, `details:text`
  - Enum values for ownership_type: `['Individual', 'Trust', 'Other']`
  - Controller with Pundit policy: admin/parent only
  - Index on ownership_type
- **Non-Functional**:
  - Validation: name and ownership_type required
  - ownership_type must be from allowed list

**Acceptance Criteria**
- OwnershipLookup model supports validations for ownership_type
- Admin/parent can create/edit/delete ownership lookups
- Non-admin users cannot access OwnershipLookup CRUD
- ownership_type enum enforced

**Model Code**
```ruby
class OwnershipLookup < ApplicationRecord
  OWNERSHIP_TYPES = %w[Individual Trust Other].freeze
  validates :name, presence: true
  validates :ownership_type, presence: true, inclusion: { in: OWNERSHIP_TYPES }
end
```

- Command: `rails g migration AddOwnershipTypeAndDetailsToOwnershipLookups ownership_type:string details:text`
