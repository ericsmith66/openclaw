### Atomic PRD-1-09: Securities Enrichment Table & Holdings Enrichment Job (FMP Primary; Finnhub Paused)

**Overview**  
Create a dedicated enrichment layer using a new `security_enrichments` table and background job to layer FMP (primary) analyst ratings, price targets, and fundamentals onto holdings.

Important change (Decision): enrichment is **per security**, not per holding.
  - Many holdings (across multiple accounts) can reference the same `security_id`.
  - We store one enrichment snapshot per `security_id` per `source`.

This enrichment is **latest snapshot only** per security per source (idempotent upsert), not historical rows.

**Log Requirements**  
Junie read `<project root>/knowledge_base/prds/prds-junie-log/junie-log-requirement.md` before implementation.

**Requirements**
- **Functional**
  - New `SecurityEnrichment` model/table: **1 row per security per source** (upserted latest snapshot).
  - Columns:
    - `security_id` string null:false
    - `source` string null:false (valid values: "fmp", "finnhub")
    - `enriched_at` datetime null:false
    - `status` string (success / partial / error / pending)
    - `data` jsonb
    - `notes` jsonb (array of human-readable strings)
    - timestamps
  - Composite unique index on [security_id, source]
  - GIN index on `data` column (`using: :gin`)
  - `HoldingsEnrichmentJob` (ActiveJob / SolidQueue; triggered post-sync + weekly cron + manual “Mission Control” button):
    - Scope equities (`type == "equity"`)
    - Normalize + dedup by `security_id` (and optionally exchange/mic if present):
      - Group holdings by `security_id` and pick a representative holding to obtain `ticker_symbol`.
      - Normalize: uppercase, strip whitespace, replace `.` with `-` (e.g., `BRK.B` → `BRK-B`)
      - If available, include Plaid `exchange` or `mic` in the dedup key
      - If no usable ticker: skip and log note "no ticker"
    - Call FMP (Premium plan; use non-legacy `stable` API base):
      - `https://financialmodelingprep.com/stable/profile?symbol=...` → sector/industry
      - `https://financialmodelingprep.com/stable/quote?symbol=...` → price
      - `https://financialmodelingprep.com/stable/key-metrics?symbol=...`
      - `https://financialmodelingprep.com/stable/ratios?symbol=...`
      - Store extracted keys in `data` jsonb; full raw response under `data["raw_response"]` (namespaced per endpoint)
      - Historical/time-series metrics (e.g., 30+ years) can be stored as arrays inside `data` (e.g., `historical_roe`) during the Ultimate phase
    - Finnhub fallback:
      - **Paused for now** due to daily call limits.
      - (When re-enabled) `/stock/recommendation` → store **most recent period** as a flat hash.
  - `asset_class` storage:
    - Move `asset_class` to the `holdings` table (stable derived attribute), not inside `holding_enrichments.data`.
    - Derive from Plaid fields (local rules in service), optionally refined by FMP sector/industry if Plaid is ambiguous.
  - Holding helpers:
    - `latest_enrichment_record(source = nil)`
    - `latest_enrichment_data(source = nil)`
    - `latest_consensus` (merge FMP → Finnhub)
    - `target_upside_percent`
    - `asset_class`
- **Non-Functional**
  - Use `Net::HTTP` for API calls; keys read from `Rails.application.credentials` or `ENV`:
    - `Rails.application.credentials.dig(:fmp, :api_key)`
    - `Rails.application.credentials.dig(:finnhub, :api_key)`
    - (dev fallback) `ENV['FMP_API_KEY']`, `ENV['FINNHUB_API_KEY']`
  - Rate-limit: single-symbol calls; control concurrency (10–20) + exponential backoff on 429
  - Privacy: local only; jsonb unencrypted (non-PII)
  - Errors: retry 3× with backoff, set status/notes
  - Performance: skip per security+source if existing row has `enriched_at > 7.days.ago` and `status == "success"`
  - Idempotency: upsert (update existing row or insert if none)
  - Status definition:
    - `success`: at least one source returned usable data
    - `partial`: (when Finnhub is enabled) FMP failed or incomplete, but Finnhub fallback succeeded
    - `error`: FMP failed (and Finnhub is disabled OR also failed)
    - `pending`: enqueued but not started (rare)

### Full Model Code

**app/models/holding.rb** (additions)
```ruby
class Holding < ApplicationRecord
  def latest_enrichment_record(source = nil)
    scope = SecurityEnrichment.where(security_id: security_id).order(enriched_at: :desc)
    scope = scope.where(source: source) if source
    scope.first
  end

  def latest_enrichment_data(source = nil)
    latest_enrichment_record(source)&.data || {}
  end

  def latest_consensus
    fmp = latest_enrichment_data("fmp")["analyst_consensus"]
    return fmp if fmp.present?

    finnhub_buy = latest_enrichment_data("finnhub").dig("recommendation_trend", "buy") || 0
    finnhub_buy >= 8 ? "Buy" : "Hold"
  end

  def target_upside_percent
    latest_enrichment_data("fmp")["target_upside_percent"]
  end

  def asset_class
    fmp = latest_enrichment_data("fmp")["asset_class"]
    return fmp if fmp.present?

    latest_enrichment_data["asset_class"] || "unknown"
  end
end
```

**app/models/security_enrichment.rb**

class SecurityEnrichment < ApplicationRecord

  validates :source, presence: true, inclusion: { in: %w[fmp finnhub] }
  validates :enriched_at, presence: true
end

### Steps to Set Up GIN Index

1. Generate the migration:
   ```bash
   rails g migration CreateSecurityEnrichments
   ```

2. Edit the migration file:
   ```ruby
   class CreateSecurityEnrichments < ActiveRecord::Migration[7.1]
     def change
       create_table :security_enrichments do |t|
         t.string :security_id, null: false
         t.string :source, null: false
         t.datetime :enriched_at, null: false
         t.string :status
         t.jsonb :data
         t.text :notes

         t.timestamps
       end

       add_index :security_enrichments, [:security_id, :source], unique: true
       add_index :security_enrichments, :data, using: :gin
     end
   end
   ```

3. Run the migration:
   ```bash
   rails db:migrate
   ```

4. Verify the index exists:
   ```bash
   rails dbconsole
   \d security_enrichments
   ```
  - Look for index on `data` using `gin`.

5. Test a sample query in Rails console:
   ```ruby
   explain = SecurityEnrichment
     .where("data @> ?", { asset_class: "equity" }.to_json)
     .explain
   puts explain
   ```
  - Should show an index scan using the GIN index on `security_enrichments.data` (planner-dependent).
