### PRD-1-10: Basic UI for Other Income CRUD (User-Maintainable)

**Overview**
New page/controller for user to manage their own OtherIncome records.

This is for **non-Plaid-syncable income sources** (side gigs, rentals, interest from unlinked accounts, bonuses, royalties, etc.).

**Requirements**
- **Functional**:
  - Route: `/other_incomes`
  - Actions: index, new, create, edit, update, destroy (standard Rails scaffold-style pages; no inline edit/modals)
  - Scope: explicit scoping via `current_user.other_incomes` for all actions
    - Unauthorized access should raise `ActiveRecord::RecordNotFound` (404) via scoped `.find`
  - Index:
    - Table view with add/edit/delete buttons
    - Show an **estimated annual total** above the table
      - Annualization rules:
        - `annual` = amount
        - `monthly` = amount × 12
        - `quarterly` = amount × 4
        - `one_time` = amount
    - Sorting via query params (no filtering/pagination in this PRD):
      - Default: `name` ascending
      - `?sort=amount_desc`
      - `?sort=updated_at_desc`
  - Link from dashboard/sidebar labeled: **"Other Income Sources"** (visible to all signed-in users)
- **Non-Functional**:
  - HTML-only (no Turbo Streams / JSON in this PRD)
  - Simple forms (no admin restriction)
  - DaisyUI styling:
    - Index table: `table table-zebra`
    - Buttons: `btn btn-primary` (New), `btn btn-outline btn-sm` (Edit/Delete)
    - Inputs: `input input-bordered`, `select select-bordered`, `textarea textarea-bordered`
  - Amounts stored as `decimal(12,2)` and displayed/input as USD dollars (no cents conversion yet)

**Data Model (OtherIncome)**
- `name` (string, required, max 100)
- `amount` (decimal(12,2), required, >= 0)
- `frequency` (string/enum, required): `annual`, `monthly`, `quarterly`, `one_time`
- `start_date` (date, optional)
- `end_date` (date, optional; if present must be >= `start_date`)
- `category` (string/enum, optional): `employment_side`, `interest_dividends`, `rental`, `capital_gains`, `pension`, `other`
- `taxable` (boolean, optional, default true)
- `notes` (text, optional)

**Acceptance Criteria**
- User can see list of their income sources
- User can add/edit/delete records
- No access to other users' records
  - Access attempts to other users' records return 404
- Delete prompts a browser-native confirm dialog
- Success flash copy:
  - Create: "Other income source added successfully."
  - Update: "Other income source updated."
  - Destroy: "Other income source removed."

**Workflow**
- Branch: `feature/prd-1-10-ui-other-income`
- Command: `rails g controller OtherIncomes index new create edit update destroy`
