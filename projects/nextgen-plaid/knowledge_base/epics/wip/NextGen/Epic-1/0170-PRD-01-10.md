**PRD-0170 v2: Add First-Class Columns to SecurityEnrichment for Critical FMP Fields**

**Overview**  
Add dedicated, typed columns to the `security_enrichments` table for the most frequently used FMP fields (e.g., price, market_cap, sector, company_name, etc.) to enable fast, indexed queries and avoid repeated JSONB `dig` operations in services, jobs, and AI prompts. The full raw FMP response remains in `data` jsonb for completeness and future-proofing.

**Requirements**
- **Functional**:
    - **Phase 1 Migration** (Add Columns): Add the following 22 columns to `security_enrichments` (all nullable, with appropriate types):
        - `price`: decimal(18,6) — current quote price
        - `market_cap`: bigint — market capitalization
        - `sector`: string — company sector
        - `industry`: string — company industry
        - `company_name`: string — full company name
        - `website`: string — company website
        - `description`: text — company description (store full text)
        - `image_url`: string — logo URL
        - `change_percentage`: decimal(10,4) — daily % change
        - `dividend_yield`: decimal(10,6) — trailing dividend yield
        - `pe_ratio`: decimal(12,4) — price-to-earnings
        - `price_to_book`: decimal(12,4) — price-to-book ratio
        - `net_profit_margin`: decimal(10,6) — net profit margin
        - `dividend_per_share`: decimal(10,4) — dividend per share
        - `free_cash_flow_yield`: decimal(10,6) — FCF yield
        - `roe`: decimal(10,6) — return on equity (short name per feedback)
        - `roa`: decimal(10,6) — return on assets (short name per feedback)
        - `beta`: decimal(10,6) — stock beta (added per feedback)
        - `roic`: decimal(10,6) — return on invested capital (added per feedback)
        - `current_ratio`: decimal(10,4) — current ratio
        - `debt_to_equity`: decimal(10,4) — debt-to-equity
        - `status`: string — enrichment status (success, error, pending, etc.)
    - **Phase 2 Migration** (Backfill): Populate new columns from existing `data` jsonb using flattened keys first, fall back to nested `raw_response` if missing. Batched with `in_batches(of: 500)` for performance.
    - **Phase 3 Migration** (Add Indexes): Create indexes on high-query fields:
        - Single-column indexes: `price`, `market_cap`, `roe`, `pe_ratio`, `sector`, `industry`, `status`
        - Compound indexes: `[:sector, :status]`, `[:industry, :status]`
        - GIN trigram index on `company_name` for full-text search
    - **Model Changes**:
        - Remove overlapping `store_accessor` declarations for these 22 fields (avoid duplication)
        - Keep typed helper methods as simple aliases: `price_d`, `market_cap_i`, etc.
    - **Service Update**: Update `FmpEnricherService#create_or_update!` to populate new columns when creating/updating enrichment records
    - No change to `data` jsonb—it remains the full raw source for future-proofing
- **Non-functional**:
    - Zero-downtime: Nullable columns first, backfill in batches, then add indexes in separate migration
    - Performance: New columns enable direct SQL queries (e.g., `where(sector: 'Technology')`) and faster Holding joins
    - Privacy: Fields are public market data—no encryption needed
    - Idempotency: Backfill can be re-run safely (uses `update_columns` to skip callbacks)

class BackfillSecurityEnrichmentColumns < ActiveRecord::Migration[8.0]

    SecurityEnrichment.unscoped.in_batches(of: 500) do |batch|

        # Flattened keys first (from FmpEnricherService), fall back to raw_response
        data = enrichment.data || {}
        raw = data['raw_response'] || {}

        # Extract nested structures
        profile = raw['profile'].is_a?(Array) ? raw['profile'].first : raw['profile']
        quote = raw['quote'].is_a?(Array) ? raw['quote'].first : raw['quote']
        key_metrics = raw['key_metrics'].is_a?(Array) ? raw['key_metrics'].first : raw['key_metrics']
        ratios = raw['ratios'].is_a?(Array) ? raw['ratios'].first : raw['ratios']

        enrichment.update_columns(

          market_cap: data['market_cap'] || profile&.dig('mktCap') || quote&.dig('marketCap'),

    add_index :security_enrichments, :roe, algorithm: :concurrently

    add_index :security_enrichments, :sector, algorithm: :concurrently
    add_index :security_enrichments, :industry, algorithm: :concurrently
    add_index :security_enrichments, :status, algorithm: :concurrently
    add_index :security_enrichments, :beta, algorithm: :concurrently

    # Compound indexes for common query patterns
    add_index :security_enrichments, [:sector, :status], algorithm: :concurrently
    add_index :security_enrichments, [:industry, :status], algorithm: :concurrently

    # GIN trigram index for full-text search on company_name

      CREATE INDEX CONCURRENTLY index_security_enrichments_on_company_name_trgm
