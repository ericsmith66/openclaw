### PRD-1-08.5: Account Sharing & Parent-Child Relationship UI

**Overview**
Create UI components to enable parents/admins to:
1. Define and manage parent-child relationships between users
2. Share/exclude specific accounts with child users
3. View and modify existing sharing permissions

## Requirements

### Functional Requirements

#### 1. Parent-Child Relationship Management

**User Relationship Setup Screen**
- Admin/parent can designate users as "parent" or "child"
- Parent user can add multiple child users
- Display parent-child relationships in a tree or list view
- Actions:
  - **Add Child**: Select from existing users or invite new user as child
  - **Remove Child**: Soft-delete the parent-child relationship
  - **View Child's Shared Accounts**: Navigate to account sharing management

**Data Model**
```ruby
# User model extension
has_many :children, class_name: "User", foreign_key: "parent_id"
belongs_to :parent, class_name: "User", optional: true

# Schema addition
add_column :users, :parent_id, :bigint
add_index :users, :parent_id
add_foreign_key :users, :users, column: :parent_id
```

#### 2. Account Sharing Management Screen

**Account Sharing Dashboard** (Parent View)
- **Context**: Parent is viewing their own accounts to share with children
- **Layout**:
  - Left panel: List of parent's accounts (from Plaid)
    - Account name, type, institution, current balance
    - Visual indicator: shared/not shared status per child
  - Right panel: List of parent's children
    - Child name, email
    - Count of shared accounts with each child
- **Actions**:
  - **Share Account with Child**:
    - Multi-select accounts
    - Select child recipient(s)
    - Create `AccountSharing` records
  - **Revoke Sharing**:
    - Remove access (soft delete `AccountSharing` record)
  - **Bulk Operations**:
    - Share all accounts with child
    - Revoke all shares from child

**Shared Accounts View** (Child View)
- **Context**: Child viewing accounts shared by parent
- **Layout**:
  - List of shared accounts (read-only)
  - Display "Shared by [Parent Name]" badge
  - Show holdings, transactions, balances for shared accounts
- **Restrictions**:
  - No edit/delete capabilities
  - Cannot see parent's unshared accounts
  - Cannot share accounts further (no cascading)

#### 3. Account Sharing Matrix (Advanced View)

**Matrix Grid**
- Rows: Parent's accounts
- Columns: Parent's children
- Cells: Checkbox (shared/not shared)
- Enable quick bulk sharing/revoking across multiple accounts/children

**Example**:
```
                    Child 1    Child 2    Child 3
Chase Checking        ✓          ✓
Chase Savings         ✓                     ✓
Vanguard Brokerage               ✓          ✓
```

#### 4. Notifications & Audit Trail

**Notifications**
- Notify child when parent shares new account
- Notify parent when sharing is successfully enabled/revoked

**Audit Log**
- Track all sharing actions:
  - Who shared what account with whom
  - Timestamp
  - Action (created/revoked)
- Display in parent dashboard

---

### Non-Functional Requirements

#### Security & Privacy
- **Authorization**:
  - Only parent or admin can manage sharing for that parent's accounts
  - Child cannot modify sharing permissions
  - Child cannot see unshared accounts or data
- **Validation**:
  - Cannot share accounts parent doesn't own
  - Cannot add child to multiple parents (single parent only)
  - Cannot create circular relationships (child cannot be parent of their parent)

#### UX/UI
- **Responsive**: Mobile-friendly account sharing management
- **Clear Visual Hierarchy**:
  - Shared accounts clearly marked with badges/icons
  - Disabled state for revoked shares (grayed out)
- **Confirmation Dialogs**:
  - Confirm before revoking access to account
  - Warning if revoking access will hide transactions/holdings child currently sees

#### Performance
- Paginate account lists for parents with many accounts
- Lazy load shared transactions/holdings in child view
- Cache account sharing status to avoid repeated queries

## User Stories

### Story 1: Parent Adds Child User
**As a parent**, I want to add my teenager as a child user, so I can share my investment accounts with them for educational purposes.

**Acceptance Criteria**:
- Parent navigates to "Manage Family" or "User Relationships"
- Parent clicks "Add Child"
- Parent enters child's email (or selects existing user)
- System sends invitation email to child (if new user)
- System creates `parent_id` relationship in `users` table
- Parent sees child listed in their "Children" section

### Story 2: Parent Shares Account with Child
**As a parent**, I want to share my Vanguard brokerage account with my child, so they can track the performance of specific investments.

**Acceptance Criteria**:
- Parent navigates to "Account Sharing" dashboard
- Parent selects "Vanguard Brokerage" account
- Parent clicks "Share with Child"
- Parent selects child from dropdown
- System creates `AccountSharing` record (parent_id, child_id, account_id)
- Child receives notification
- Child sees "Vanguard Brokerage" in their shared accounts list

### Story 3: Child Views Shared Account
**As a child**, I want to view the account my parent shared with me, so I can see the holdings and transactions.

**Acceptance Criteria**:
- Child logs in and navigates to "Accounts"
- Child sees shared account with "Shared by [Parent Name]" badge
- Child clicks account and sees:
  - Holdings associated with that account
  - Transactions associated with that account
  - Account balance snapshots
- Child cannot edit or delete account
- Child does NOT see parent's other unshared accounts

### Story 4: Parent Revokes Account Sharing
**As a parent**, I want to revoke my child's access to a shared account, so they can no longer see that account's data.

**Acceptance Criteria**:
- Parent navigates to "Account Sharing" dashboard
- Parent selects shared account and clicks "Revoke Access" for specific child
- System soft-deletes `AccountSharing` record
- Child receives notification
- Child can no longer see account in their shared accounts list
- Child can no longer see holdings/transactions for that account

### Story 5: Admin Views All Parent-Child Relationships
**As an admin**, I want to see all parent-child relationships in the system, so I can audit and troubleshoot sharing issues.

**Acceptance Criteria**:
- Admin navigates to "Admin → User Relationships"
- Admin sees list of all users with `parent_id` set
- Admin can filter by parent or child
- Admin can view all `AccountSharing` records for any parent-child pair
- Admin can manually create/delete relationships (with audit log)

## UI Component Specifications

### 1. Parent-Child Relationship Manager

**Route**: `/family` or `/relationships`

**Components**:
- `ParentChildList.tsx`: Display parent's children in card or table format
- `AddChildModal.tsx`: Form to add child (email input, user search)
- `RemoveChildConfirmation.tsx`: Confirmation dialog before removing relationship

**API Endpoints**:
```ruby
# Parent-child relationship management
POST   /api/v1/users/:parent_id/children        # Add child
DELETE /api/v1/users/:parent_id/children/:id    # Remove child
GET    /api/v1/users/:parent_id/children        # List children
```

### 2. Account Sharing Dashboard

**Route**: `/accounts/sharing`

**Components**:
- `AccountSharingDashboard.tsx`: Main container with two-panel layout
- `AccountListPanel.tsx`: Parent's accounts with share status indicators
- `ChildListPanel.tsx`: Parent's children with share counts
- `ShareAccountModal.tsx`: Multi-select accounts + select child(ren)
- `SharingMatrix.tsx`: Grid view for bulk sharing (advanced mode)

**API Endpoints**:
```ruby
# Account sharing management
POST   /api/v1/account_sharings                       # Share account(s)
DELETE /api/v1/account_sharings/:id                   # Revoke sharing
GET    /api/v1/users/:parent_id/account_sharings     # List all shares for parent
GET    /api/v1/accounts/:account_id/sharings         # List children account is shared with
```

### 3. Shared Accounts View (Child)

**Route**: `/accounts/shared`

**Components**:
- `SharedAccountsList.tsx`: Read-only list of accounts shared by parent
- `SharedAccountDetail.tsx`: Holdings, transactions, balances for shared account
- `SharedByBadge.tsx`: Badge showing "Shared by [Parent Name]"

**API Endpoints**:
```ruby
# Child viewing shared accounts
GET /api/v1/users/:child_id/shared_accounts           # Accounts shared with child
GET /api/v1/accounts/:account_id/shared_holdings      # Holdings for shared account
GET /api/v1/accounts/:account_id/shared_transactions  # Transactions for shared account
```

### 4. Notifications

**Components**:
- `SharingNotification.tsx`: Toast/banner when account is shared or revoked
- `NotificationList.tsx`: List of all sharing-related notifications

**Events**:
- `account.shared`: Parent shared account with child
- `account.revoked`: Parent revoked child's access to account
- `child.added`: Parent added child user

## Data Queries & Filtering

### Parent Account Query (with Sharing Status)
```ruby
# Controller: AccountsController
def index_with_sharing_status
  accounts = current_user.accounts.includes(:account_sharings)

  accounts.map do |account|
    {
      id: account.id,
      name: account.name,
      balance: account.current_balance,
      shared_with: account.account_sharings.includes(:child).map do |sharing|
        { child_id: sharing.child_id, child_name: sharing.child.name }
      end
    }
  end
end
```

### Child Shared Account Query
```ruby
# Controller: SharedAccountsController
def index
  # Only show accounts explicitly shared via AccountSharing
  shared_account_ids = AccountSharing
    .where(child_id: current_user.id)
    .where(deleted_at: nil)  # Only active shares
    .pluck(:account_id)

  accounts = Account
    .where(id: shared_account_ids)
    .includes(:plaid_item, :holdings, :transactions)
end
```

### Child Holdings Query (Filtered by Shared Accounts)
```ruby
# Model: Holding
scope :visible_to_child, ->(child_user) {
  shared_account_ids = AccountSharing
    .where(child_id: child_user.id)
    .where(deleted_at: nil)
    .pluck(:account_id)

  where(account_id: shared_account_ids)
}
```

### Database Schema Changes

```ruby
  def change

    add_index :account_sharings, :deleted_at
  end

```

### Security Middleware
```ruby
# app/controllers/concerns/account_sharing_authorization.rb
module AccountSharingAuthorization
  extend ActiveSupport::Concern

  def authorize_parent_access!(account)
    unless account.user_id == current_user.id
      raise Pundit::NotAuthorizedError, "You can only share your own accounts"
    end
  end

  def authorize_child_access!(account)
    shared = AccountSharing.exists?(
      child_id: current_user.id,
      account_id: account.id,
      deleted_at: nil
    )

    unless shared
      raise Pundit::NotAuthorizedError, "This account is not shared with you"
    end
  end
end
```

## Out of Scope

- **Cascade sharing**: Child cannot share parent's account with another user
- **Bidirectional sharing**: Child cannot share accounts back to parent
- **Granular permissions**: All-or-nothing sharing (no read-only holdings vs transactions)
- **Time-limited sharing**: No expiration dates on sharing
- **Sharing history UI**: Audit log visible in admin only (not parent UI)
