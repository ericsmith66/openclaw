# PRD-0160: Enable STI on Account and Transaction Models

## Overview
Two atomic PRDs to enable proper Single Table Inheritance (STI) on Account and Transaction models. This removes architectural inconsistency, enables type-specific processing, and prepares for future subclass implementations.

**Implementation Order**: PRD-0160.01 (Account) must complete before PRD-0160.02 (Transaction).

---

## PRD-0160.01: Enable STI on Account Model with plaid_account_type Preservation

### Overview
Enable standard Rails STI on the `Account` model by repurposing the `type` column for inheritance while preserving raw Plaid account type data in a new `plaid_account_type` column. This removes the current STI disable (`self.inheritance_column = :_type_disabled`), ensures consistency for future Transaction STI, and prepares for potential Account subclasses (e.g., `CreditAccount`) without immediate behavioral changes.

## PRD-0160.02: Introduce STI on Transaction Model with Subclasses

```sql
-- 1. Distribution of proposed transaction types
SELECT
  CASE
    WHEN investment_transaction_id IS NOT NULL THEN 'InvestmentTransaction'
    WHEN investment_type IS NOT NULL THEN 'InvestmentTransaction'
```

   ```ruby
     # Hard dependency - fail fast if Account STI not deployed

           WHEN account_id IS NULL THEN 'RegularTransaction'
   ```

   ```ruby
   def determine_transaction_class(account, txn)
     if txn.investment_transaction_id.present? || txn.investment_type.present?
       InvestmentTransaction
     elsif account.plaid_account_type == 'credit' || account.subtype&.include?('credit')
       CreditTransaction
     else
       RegularTransaction
     end
   end

   def sync_transaction(account, plaid_txn)
     klass = determine_transaction_class(account, plaid_txn)
     transaction = Transaction.unscoped.find_by(
       account: account,
       transaction_id: plaid_txn.transaction_id
     )

     if transaction.present?
       if transaction.class != klass
         handle_type_change(transaction, klass, account, plaid_txn)
       else
         transaction.assign_attributes(plaid_txn_to_attrs(plaid_txn))
         transaction.save!
       end
     else
       transaction = klass.new(plaid_txn_to_attrs(plaid_txn))
       transaction.account = account
       transaction.save!
     end
   end

   def handle_type_change(old_txn, new_klass, account, plaid_txn)
     # Soft delete old
     old_txn.update!(deleted_at: Time.current)

     # Create new with correct type
     new_txn = new_klass.create!(plaid_txn_to_attrs(plaid_txn).merge(account: account))

     # Log correction (silent - no user notification)
     TransactionCorrection.create_from_plaid_correction!(old_txn, new_txn, plaid_txn.to_h)

     Rails.logger.warn("Transaction type changed: #{old_txn.id} (#{old_txn.class}) → #{new_txn.id} (#{new_klass})")
   end
   ```

```ruby
# app/models/transaction.rb (base)
class Transaction < ApplicationRecord
  # STI enabled (Rails uses 'type' column automatically)

  belongs_to :account
  has_one :enriched_transaction, dependent: :destroy

  # Shared associations
  belongs_to :merchant, optional: true
  belongs_to :personal_finance_category, optional: true
  belongs_to :transaction_code, optional: true

  # Shared validations
  validates :date, presence: true
  validates :amount, presence: true
  validates :transaction_id, presence: true, if: -> { source == "plaid" }

  # Type immutability
  validate :type_immutable, on: :update

  # Validation: investment-specific fields only on InvestmentTransaction
  validate :no_investment_fields_on_non_investment

  # Shared enums
  enum :source, { plaid: "plaid", manual: "manual", csv: "csv" }

  # STI-based scopes
  scope :investments, -> { where(type: 'InvestmentTransaction') }
  scope :credits, -> { where(type: 'CreditTransaction') }
  scope :regulars, -> { where(type: 'RegularTransaction') }

  # Backwards compatibility alias (remove after 3 months)
  scope :investment, -> { where(type: 'InvestmentTransaction') }

  default_scope { where(deleted_at: nil) }

  private

  def type_immutable
    if type_changed?
      errors.add(:type, "cannot be changed once set")
    end
  end

  def no_investment_fields_on_non_investment
    if dividend_type.present? && !is_a?(InvestmentTransaction)
      errors.add(:dividend_type, "is only for investment transactions")
    end
  end

  def jsonb_payload_size_limit
    limit_bytes = 1_000_000
    blobs = [location, payment_meta, counterparties]
    total = blobs.compact.sum { |h| h.to_json.bytesize }
    errors.add(:base, "JSON payload too large (#{total} bytes > #{limit_bytes})") if total > limit_bytes
  end
end
```

### Rollback Strategy

If issues arise after deployment:

1. Run SQL to clear type column:
   ```sql
   UPDATE transactions SET type = NULL;
   ```

   ```bash
   rm app/models/investment_transaction.rb
   rm app/models/credit_transaction.rb
   rm app/models/regular_transaction.rb

   ```

