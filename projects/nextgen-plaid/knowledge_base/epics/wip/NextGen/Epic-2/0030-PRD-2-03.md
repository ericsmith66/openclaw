# PRD-2-03: Snapshot – Asset Allocation Breakdown

## Overview
Extend FinancialSnapshotJob JSON with asset allocation percentages by class (equities, fixed income, cash, alternative) from Position data via DataProvider. Directly feeds Epic 3 asset allocation page pie/bar components.

## Status
**Branch**: `feature/epic-2-financial-snapshots`
**Dependencies**: PRD-2-02 (FinancialSnapshotJob core)

## Requirements

### Functional
- Extend `DataProvider#asset_allocation_breakdown` (if not already complete from PRD-2-01b)
- Group `Position.current_value` by `asset_class` ('equity', 'fixed_income', 'cash', 'alternative')
- Return percentages as decimals 0–1, sum forced to exactly 1.0 (tolerance 0.0001)
- Fallback: 'other' bucket for nil/uncategorized asset_class
- Use `holding.data` JSON if SecurityEnrichment `asset_class` column missing
- JSON structure per Target Snapshot JSON Structure v1

### Non-Functional
- Accurate to 4 decimals, round and adjust largest for sum=1.0
- Handle zero positions: return `{}` (empty hash)

## Acceptance Criteria
1. JSON `asset_allocation` sums to 1.0 within 0.0001
2. Matches Position aggregates grouped by asset_class
3. Handles no positions (empty hash `{}`)
4. Job runtime unchanged (<5s)
5. Fallback to holding.data when enrichment nil
6. Snapshot includes valid allocation data

## Test Cases
```ruby
it "includes asset allocation summing to 1.0" do
  create_list(:position, 5, user: user, asset_class: 'equity', current_value: 100_000)
  create_list(:position, 3, user: user, asset_class: 'cash', current_value: 50_000)

  FinancialSnapshotJob.perform_now(user)
  snapshot = user.financial_snapshots.last

  allocation = snapshot.data['asset_allocation']
  expect(allocation['equity']).to be_within(0.01).of(0.769) # 500k/650k
  expect(allocation['cash']).to be_within(0.01).of(0.231)   # 150k/650k
  expect(allocation.values.sum).to be_within(0.0001).of(1.0)
end
```
