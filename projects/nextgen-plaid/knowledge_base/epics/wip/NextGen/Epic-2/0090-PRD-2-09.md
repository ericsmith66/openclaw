# PRD-2-09: Net Worth Dashboard Layout

## Overview
Implement the net worth dashboard UI using snapshot data, integrating placeholders from PRD-2-00 with real aggregates via DataProvider. Delivers the v1 user-facing pages.

## Status
**Branch**: `feature/epic-2-financial-snapshots`
**Dependencies**: PRD-2-00 (Scaffold ✅), PRD-2-02+ (Snapshot data)

## Requirements

### Functional
- **Controller**: `NetWorth::DashboardController#index`
  - Fetch latest complete snapshot: `FinancialSnapshot.latest_for_user(current_user)`
  - Fallback: Use `DataProvider.new(current_user).build_snapshot_hash` if no snapshot
  - Pass data to view

- **Components**: ViewComponent classes
  - `NetWorthHeroComponent` - Total NW + deltas (large card)
  - `AssetAllocationChartComponent` - Pie/bar chart for allocation
  - `SectorWeightsComponent` - Sector breakdown (null-safe)
  - `RecentActivityComponent` - Transaction summary
  - `PerformancePlaceholderComponent` - Trend chart placeholder

- **Routes**: Build on PRD-2-00 `/net_worth/dashboard`

- **Data**: Query snapshot JSON directly (no real-time calcs)

- **Placeholder Handling**: If no snapshot, show message: "Generating your first snapshot. Check back soon!"

### Non-Functional
- **UI**: Tailwind/DaisyUI, professional theme for 22-30 audience
- **Performance**: No real-time calculations; JSON direct read
- **Responsive**: Mobile-friendly layout

## Acceptance Criteria
1. Dashboard renders `total_net_worth`, deltas from snapshot/DataProvider
2. Allocation pie chart matches JSON percentages
3. Sector weights display correctly (handle null)
4. Feature flag gates access (from PRD-2-00)
5. Handles no snapshot gracefully (placeholder message)
6. Capybara smoke test passes

## Workflow

### Setup
1. Ensure PRD-2-00 scaffold complete
2. Ensure snapshots generating (PRD-2-02+)

### Implementation Steps

1. Generate controller if not exists:
   ```bash
   rails g controller net_worth/dashboard index
   ```

2. Generate ViewComponents:
   ```bash
   rails g component NetWorthHero
   rails g component AssetAllocationChart
   rails g component SectorWeights
   rails g component RecentActivity
   rails g component PerformancePlaceholder
   ```

3. Implement controller:
   ```ruby
   class NetWorth::DashboardController < ApplicationController
     before_action :authenticate_user!
     before_action :require_new_layout!

     def index
       @snapshot = FinancialSnapshot.latest_for_user(current_user)

       @data = if @snapshot
                 @snapshot.data.with_indifferent_access
               else
                 # Fallback: Generate on-the-fly
                 Reporting::DataProvider.new(current_user).build_snapshot_hash.with_indifferent_access
               end
     end

     private

     def require_new_layout!
       redirect_to root_path unless ENV['ENABLE_NEW_LAYOUT'] == 'true'
     end
   end
   ```

4. Implement components (example):
   ```ruby
   # app/components/net_worth_hero_component.rb
   class NetWorthHeroComponent < ViewComponent::Base
     def initialize(data:)
       @data = data
     end

     def total_net_worth
       number_to_currency(@data[:total_net_worth] || 0)
     end

     def delta_day
       @data[:delta_day] || 0
     end

     def delta_class
       delta_day >= 0 ? 'text-success' : 'text-error'
     end
   end
   ```

5. Create views:
   ```erb
   <!-- app/views/net_worth/dashboard/index.html.erb -->
   <div class="container mx-auto p-6">
     <% if @snapshot %>
       <%= render NetWorthHeroComponent.new(data: @data) %>
       <%= render AssetAllocationChartComponent.new(allocation: @data[:asset_allocation]) %>
       <%= render SectorWeightsComponent.new(weights: @data[:sector_weights]) %>
       <%= render RecentActivityComponent.new(summary: @data[:monthly_transaction_summary]) %>
       <%= render PerformancePlaceholderComponent.new(history: @data[:historical_net_worth]) %>

       <% if @snapshot.stale? %>
         <div class="alert alert-warning">Data may be outdated. Last sync: <%= @snapshot.updated_at.to_s(:short) %></div>
       <% end %>
     <% else %>
       <div class="hero min-h-screen">
         <div class="text-center">
           <h1 class="text-4xl">Generating your first snapshot</h1>
           <p class="text-lg text-gray-500">Check back soon!</p>
         </div>
       </div>
     <% end %>
   </div>
   ```

6. Add chart library if needed (e.g., Chartkick):
   ```ruby
   # Gemfile
   gem 'chartkick'
   ```

7. Write specs (controller, feature, components)

8. Manual test:
   - With snapshot data
   - Without snapshot
   - Stale snapshot
   - Feature flag off

9. Commit: `feat(PRD-2-09): Implement net worth dashboard UI with snapshot data`

## Lifecycle Integration Test

Add to this PRD per feedback:

```ruby
# spec/features/snapshot_lifecycle_spec.rb
require 'rails_helper'

RSpec.describe "Full Snapshot Lifecycle", type: :feature do
  let(:user) { create(:user) }
  let(:admin) { create(:user, role: :admin) }

  it "generates, validates, exports, and displays snapshot end-to-end" do
    # 1. Seed Plaid data
    create_list(:position, 10, user: user, asset_class: 'equity', current_value: 50_000)
    create(:account, user: user, current_balance: 100_000)
    create(:item, user: user, last_successful_update: 1.hour.ago)

    # 2. Run job
    FinancialSnapshotJob.perform_now(user)

    # 3. Verify snapshot
    snapshot = user.financial_snapshots.last
    expect(snapshot.status).to eq('complete')
    expect(snapshot.data['total_net_worth']).to eq(600_000)

    # 4. Admin views
    sign_in admin
    visit admin_snapshot_path(snapshot)
    expect(page).to have_content(snapshot.data['total_net_worth'])

    # 5. User downloads
    sign_in user
    visit api_snapshot_download_path(snapshot)
    json = JSON.parse(page.body)
    expect(json['total_net_worth']).to eq(600_000)

    # 6. User views dashboard
    visit net_worth_dashboard_path
    expect(page).to have_content("$600,000")
    expect(page).to have_content("Asset Allocation")
  end
end
```
