# PRD-2-01: FinancialSnapshot Model & Migration

## Overview
Create FinancialSnapshot model to store daily JSON blobs per user, serving as the primary data source for Epic 3 UI components like net worth totals and allocation breakdowns. This establishes the foundation for all subsequent snapshot aggregations.

## Requirements

### Functional
- **Model**: `FinancialSnapshot` with:
  - Associations: `belongs_to :user`
  - Columns:
    - `snapshot_at:datetime` - Unique per user/CST-day, use `APP_TIMEZONE` constant
    - `data:jsonb` - For aggregates (reference Target Snapshot JSON Structure v1 from overview)
    - `status:enum` - Values: `:pending`, `:complete`, `:error`, `:stale`, `:empty`, `:rolled_back`
    - `schema_version:integer` - Presence required, inclusion 1..2
  - **Timezone normalization**: `before_validation` callback to normalize `snapshot_at` to beginning_of_day in CST

- **Migration**: Create table with indexes:
  - Standard index on `user_id`
  - Composite index on `[:user_id, :snapshot_at]` (descending order for recent queries)
  - Functional unique index: `user_id, DATE(snapshot_at AT TIME ZONE 'America/Chicago')` to enforce one snapshot per user per CST day
  - GIN index on `data` for JSONB queries

- **Basic scopes**:
  - `latest_for_user(user)` - Returns most recent complete snapshot for user
  - `for_date_range(user, start_date, end_date)` - Returns snapshots within date range
  - `recent_for_user(user, days = 30)` - Returns last N days of snapshots

- **Data quality methods**:
  - `data_quality_score` - Calculated via `Reporting::DataQualityValidator` (create in this PRD or PRD-2-02)
  - `warnings` - Array extracted from `data['data_quality']['warnings']`

- **Rollback method**:
  - `self.rollback_to_date(user, date)` - Soft delete by updating status to `:rolled_back` for snapshots after specified date

- **Schema adapter**:
  - Create `Reporting::SnapshotAdapter` module with `read_v1(data)` method (returns data as-is for now)
  - Stub for future version handling

### Non-Functional
- JSONb optimized for quick reads (Epic 3 UI will query `data` directly)
- Privacy: Application-level scoping — users access only own snapshots via `current_user.financial_snapshots`
- No job yet — just storage setup

## Architectural Context
- **Rails MVC**: Model only (no controller/view in this PRD)
- **PostgreSQL**: Use JSONb for flexible structure
- **Integration**: Future job will populate; assume synced data in Position/Account/Liability models
- **Timezone**: Use `APP_TIMEZONE` constant from `config/initializers/constants.rb`
- **Reference**: No UI, but future admin previews follow `knowledge_base/UI/STYLE_GUIDE.md` and `knowledge_base/UI/templates/general.md`

## Acceptance Criteria
1. Running migration creates `financial_snapshots` table with correct columns/indexes
2. `FinancialSnapshot.create(user: user, snapshot_at: Date.current, data: {}, schema_version: 1)` succeeds
3. Unique constraint enforces one snapshot per user per CST day (test with two records on same CST date but different UTC — expect uniqueness violation)
4. Scope `FinancialSnapshot.latest_for_user(user)` returns most recent complete snapshot
5. Scope `FinancialSnapshot.recent_for_user(user, 7)` returns last 7 days
6. Application-level scoping: Non-owner user query returns empty (not error)
7. Enum status defaults to `:pending`
8. JSONb field accepts hash and stores correctly
9. `schema_version` validates inclusion in 1..2
10. `data_quality_score` method exists (may delegate to validator)
11. `rollback_to_date` updates matching snapshots to `:rolled_back` status
12. `Reporting::SnapshotAdapter.read_v1(data)` returns data unchanged
13. Timezone normalization: `snapshot_at` set to beginning_of_day in CST before validation

## Test Cases

### Model Spec (RSpec)

```ruby
require 'rails_helper'

RSpec.describe FinancialSnapshot, type: :model do
  let(:user) { create(:user) }
  let(:other_user) { create(:user) }

  describe 'validations' do
    it 'validates unique snapshot_at per user per CST day' do
      # Same CST day, different UTC times
      time1 = Time.zone.parse('2026-01-24 01:00:00 CST')
      time2 = Time.zone.parse('2026-01-24 23:00:00 CST')

      create(:financial_snapshot, user: user, snapshot_at: time1)
      expect {
        create(:financial_snapshot, user: user, snapshot_at: time2)
      }.to raise_error(ActiveRecord::RecordNotUnique)
    end

    it 'allows same CST day for different users' do
      time = Time.zone.parse('2026-01-24 12:00:00 CST')
      expect {
        create(:financial_snapshot, user: user, snapshot_at: time)
        create(:financial_snapshot, user: other_user, snapshot_at: time)
      }.not_to raise_error
    end

    it 'validates schema_version inclusion' do
      expect {
        create(:financial_snapshot, user: user, schema_version: 3)
      }.to raise_error(ActiveRecord::RecordInvalid, /schema_version/)
    end

    it 'requires schema_version presence' do
      snapshot = build(:financial_snapshot, user: user, schema_version: nil)
      expect(snapshot).not_to be_valid
      expect(snapshot.errors[:schema_version]).to include("can't be blank")
    end
  end

  describe 'timezone normalization' do
    it 'normalizes snapshot_at to beginning of day in CST' do
      time = Time.zone.parse('2026-01-24 15:30:45 CST')
      snapshot = create(:financial_snapshot, user: user, snapshot_at: time)

      expect(snapshot.snapshot_at).to eq(Time.zone.parse('2026-01-24 00:00:00 CST'))
    end
  end

  describe 'jsonb data storage' do
    it 'stores and retrieves jsonb data correctly' do
      data = {
        total_net_worth: 1_000_000.0,
        asset_allocation: { equity: 0.6, cash: 0.4 }
      }
      snapshot = create(:financial_snapshot, user: user, data: data, schema_version: 1)

      expect(snapshot.reload.data['total_net_worth']).to eq(1_000_000.0)
      expect(snapshot.data['asset_allocation']['equity']).to eq(0.6)
    end
  end

  describe 'enums' do
    it 'defaults status to pending' do
      snapshot = create(:financial_snapshot, user: user)
      expect(snapshot.status).to eq('pending')
    end

    it 'supports all status values' do
      snapshot = create(:financial_snapshot, user: user)

      expect { snapshot.update!(status: :complete) }.not_to raise_error
      expect { snapshot.update!(status: :error) }.not_to raise_error
      expect { snapshot.update!(status: :stale) }.not_to raise_error
      expect { snapshot.update!(status: :empty) }.not_to raise_error
      expect { snapshot.update!(status: :rolled_back) }.not_to raise_error
    end
  end

  describe 'scopes' do
    before do
      @snap1 = create(:financial_snapshot, user: user, snapshot_at: 5.days.ago, status: :complete)
      @snap2 = create(:financial_snapshot, user: user, snapshot_at: 3.days.ago, status: :complete)
      @snap3 = create(:financial_snapshot, user: user, snapshot_at: 1.day.ago, status: :pending)
      @snap_other = create(:financial_snapshot, user: other_user, snapshot_at: 1.day.ago, status: :complete)
    end

    describe '.latest_for_user' do
      it 'returns most recent complete snapshot for user' do
        expect(FinancialSnapshot.latest_for_user(user)).to eq(@snap2)
      end

      it 'returns nil if no complete snapshots' do
        @snap1.update!(status: :error)
        @snap2.update!(status: :stale)
        expect(FinancialSnapshot.latest_for_user(user)).to be_nil
      end
    end

    describe '.for_date_range' do
      it 'returns snapshots within date range' do
        results = FinancialSnapshot.for_date_range(user, 4.days.ago.to_date, 2.days.ago.to_date)
        expect(results).to contain_exactly(@snap2)
      end
    end

    describe '.recent_for_user' do
      it 'returns last N days of snapshots' do
        results = FinancialSnapshot.recent_for_user(user, 4)
        expect(results).to contain_exactly(@snap2, @snap3)
      end
    end
  end

  describe 'application-level scoping' do
    it 'user can only access own snapshots via association' do
      my_snap = create(:financial_snapshot, user: user)
      other_snap = create(:financial_snapshot, user: other_user)

      expect(user.financial_snapshots).to include(my_snap)
      expect(user.financial_snapshots).not_to include(other_snap)
    end
  end

  describe '#rollback_to_date' do
    it 'marks newer snapshots as rolled_back' do
      snap1 = create(:financial_snapshot, user: user, snapshot_at: 5.days.ago, status: :complete)
      snap2 = create(:financial_snapshot, user: user, snapshot_at: 3.days.ago, status: :complete)
      snap3 = create(:financial_snapshot, user: user, snapshot_at: 1.day.ago, status: :complete)

      FinancialSnapshot.rollback_to_date(user, 4.days.ago.to_date)

      expect(snap1.reload.status).to eq('complete')
      expect(snap2.reload.status).to eq('rolled_back')
      expect(snap3.reload.status).to eq('rolled_back')
    end

    it 'only affects specified user' do
      my_snap = create(:financial_snapshot, user: user, snapshot_at: 1.day.ago, status: :complete)
      other_snap = create(:financial_snapshot, user: other_user, snapshot_at: 1.day.ago, status: :complete)

      FinancialSnapshot.rollback_to_date(user, 2.days.ago.to_date)

      expect(my_snap.reload.status).to eq('rolled_back')
      expect(other_snap.reload.status).to eq('complete')
    end
  end

  describe '#data_quality_score' do
    it 'exists and returns numeric value' do
      snapshot = create(:financial_snapshot, user: user, data: {
        data_quality: { score: 95, warnings: [] }
      })

      expect(snapshot.data_quality_score).to be_a(Numeric)
    end
  end

  describe '#warnings' do
    it 'extracts warnings array from data' do
      warnings = ['Missing price for 2 holdings', 'Stale sync']
      snapshot = create(:financial_snapshot, user: user, data: {
        data_quality: { warnings: warnings }
      })

      expect(snapshot.warnings).to eq(warnings)
    end

    it 'returns empty array when no warnings' do
      snapshot = create(:financial_snapshot, user: user, data: {})
      expect(snapshot.warnings).to eq([])
    end
  end

  describe 'edge cases' do
    it 'handles negative net worth' do
      snapshot = create(:financial_snapshot, user: user, data: {
        total_net_worth: -500_000.0
      })
      expect(snapshot.data['total_net_worth']).to eq(-500_000.0)
    end

    it 'handles very large portfolio values' do
      snapshot = create(:financial_snapshot, user: user, data: {
        total_net_worth: 1_500_000_000.00
      })
      expect(snapshot.data['total_net_worth']).to eq(1_500_000_000.00)
    end

    it 'handles leap year dates' do
      leap_date = Time.zone.parse('2024-02-29 12:00:00 CST')
      expect {
        create(:financial_snapshot, user: user, snapshot_at: leap_date)
      }.not_to raise_error
    end
  end
end
```

### Migration Spec

```ruby
require 'rails_helper'

RSpec.describe 'FinancialSnapshotMigration', type: :migration do
  it 'is reversible' do
    expect {
      migrate!
      migrate_down!
      migrate!
    }.not_to raise_error
  end

  it 'creates required indexes' do
    migrate!

    indexes = ActiveRecord::Base.connection.indexes(:financial_snapshots)

    expect(index_names).to include(match(/financial_snapshots.*data/))

  end
```

### Setup
1. Ensure on correct branch: `git checkout feature/epic-2-financial-snapshots`
2. Pull latest: `git pull origin main` (if needed)

### Implementation Steps
1. Create constants file if not exists: `config/initializers/constants.rb`
   ```ruby
   APP_TIMEZONE = 'America/Chicago'.freeze
   ```

2. Generate model: `rails g model FinancialSnapshot user:references snapshot_at:datetime data:jsonb status:integer schema_version:integer`

3. Edit migration to add:
   - Enums definition in comment
   - Composite index on `[:user_id, :snapshot_at]` with descending order
   - Functional unique index on user_id + CST date
   - GIN index on data column

4. Create `Reporting::SnapshotAdapter` module in `app/services/reporting/snapshot_adapter.rb`

5. Add model methods:
   - Scopes (latest_for_user, for_date_range, recent_for_user)
   - Instance methods (data_quality_score, warnings)
   - Class method (rollback_to_date)
   - Before validation callback for timezone normalization

6. Create factory in `spec/factories/financial_snapshots.rb`

7. Write specs (model + migration)

8. Run tests: `rspec spec/models/financial_snapshot_spec.rb`
