# PRD-2-07: Snapshot Validation & Admin Preview

## Overview
Add admin-only route to list/view snapshots and basic validation (e.g., sums check) for debugging before Epic 3, leveraging DataProvider.

## Status
**Branch**: `feature/epic-2-financial-snapshots`
**Dependencies**: PRD-2-06 (Historical Trends)

## Requirements

### Functional
- **Controller**: `Admin::SnapshotsController`
  - `index`: List recent 50 snapshots (all users), paginate if >50
  - `show`: Display pretty JSON for specific snapshot

- **Validation**: On job save, check:
  - Asset allocation sum ≈ 1.0 (tolerance 0.01)
  - Net worth > -10M (sanity check)
  - Set status to `:warning` if validation fails (new enum value if needed, or use data_quality warnings)
  - Display quality score and warnings in admin preview

- **Routes**:
  - `/admin/snapshots` (index)
  - `/admin/snapshots/:id` (show)

- **Auth**: `before_action :require_admin!`
  - Use `current_user.admin?` (requires User role enum from Epic 2 overview)

- **UI**:
  - Index: DaisyUI `table table-zebra` with columns: User, Date, Status (badge), Net Worth, Quality Score, Actions
  - Show: Pretty printed JSON in `<pre class="prose">` block
  - Non-admin: 403 with ForbiddenComponent

### Non-Functional
- Pretty print: `JSON.pretty_generate`
- Table follows `knowledge_base/UI/templates/table.md`
- Business theme per `knowledge_base/UI/STYLE_GUIDE.md`

## Acceptance Criteria
1. `/admin/snapshots` shows table with user, date, status badge, NW preview, quality score
2. `/admin/snapshots/:id` shows pretty JSON
3. Non-admin: 403 with forbidden component
4. Validation sets warnings correctly (e.g., sum off = warning in data_quality)
5. Index paginates if >50 records

## Test Cases
```ruby
describe Admin::SnapshotsController, type: :controller do
  let(:admin) { create(:user, role: :admin) }
  let(:regular_user) { create(:user, role: :intern) }

  before { sign_in admin }

  describe "GET #index" do
    it "lists recent snapshots" do
      create_list(:financial_snapshot, 10, user: regular_user)
      get :index

      expect(response).to be_successful
      expect(assigns(:snapshots).size).to eq(10)
    end

    it "denies access to non-admin" do
      sign_in regular_user
      get :index

      expect(response).to have_http_status(:forbidden)
    end
  end

  describe "GET #show" do
    it "shows pretty JSON" do
      snapshot = create(:financial_snapshot, :with_full_data, user: regular_user)
      get :show, params: { id: snapshot.id }

      expect(response.body).to include(JSON.pretty_generate(snapshot.data))
    end
  end
end
```

## Workflow
1. **Prerequisite**: Create User role migration if not exists:
   ```ruby
   add_column :users, :role, :integer, default: 0 # 0=intern, 1=advisor, 2=admin
   ```

2. Generate controller: `rails g controller admin/snapshots index show`

3. Add routes in `config/routes.rb`:
   ```ruby
   namespace :admin do
     resources :snapshots, only: [:index, :show]
   end
   ```

4. Implement controller with authorization

5. Create views:
   - `app/views/admin/snapshots/index.html.erb` (table)
   - `app/views/admin/snapshots/show.html.erb` (JSON display)

6. Add validation logic to DataProvider or FinancialSnapshotJob

7. Add specs

8. Manual test as admin and non-admin

9. Commit: `feat(PRD-2-07): Add admin snapshot validation and preview interface`
