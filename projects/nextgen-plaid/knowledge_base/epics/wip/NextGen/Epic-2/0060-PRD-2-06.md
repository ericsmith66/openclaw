# PRD-2-06: Snapshot – Historical Deltas & Trend Support

## Overview
Add historical net worth array to JSON for Epic 3 performance placeholder trends via DataProvider.

## Status
**Branch**: `feature/epic-2-financial-snapshots`
**Dependencies**: PRD-2-05 (Holdings & Transactions)

## Requirements

### Functional
- On job run, fetch last 30 complete snapshots for user
- Add to JSON: `"historical_net_worth": [{date: "YYYY-MM-DD", value: decimal}, ...]` sorted ascending by date
- Use composite index `[:user_id, :snapshot_at]` for efficient query
- Accept sparse data for first 30 days (no backfill required for v1)
- Limit to 30 days to keep JSON small

### Non-Functional
- Use `FinancialSnapshot.recent_for_user(user, 30)` scope
- Query only `snapshot_at` and `data->>'total_net_worth'` fields
- Return empty array if no history

## Acceptance Criteria
1. Historical array has up to 30 entries, sorted asc by date
2. Values match prior snapshots' total_net_worth
3. Empty array if no history
4. Uses composite index efficiently
5. Job runtime <5s

## Test Cases
```ruby
it "includes historical net worth trends" do
  5.times do |i|
    create(:financial_snapshot, user: user, snapshot_at: (i + 1).days.ago,
           data: { total_net_worth: 500_000 + (i * 10_000) }, status: :complete)
  end

  FinancialSnapshotJob.perform_now(user)
  snapshot = user.financial_snapshots.last

  history = snapshot.data['historical_net_worth']
  expect(history.size).to eq(5)
  expect(history.first['date']).to be < history.last['date']
  expect(history.last['value']).to eq(510_000)
end
```
