# PRD-2-04: Snapshot – Sector Weights

## Overview
Add equity sector weights to FinancialSnapshotJob JSON via DataProvider, powering Epic 3 sector weights page.

## Status
**Branch**: `feature/epic-2-financial-snapshots`
**Dependencies**: PRD-2-03 (Asset Allocation)

## Requirements

### Functional
- Group Position (equity only) by `sector` column, sum `current_value`, normalize to % of equities total
- Return percentages 0–1, sum to 1.0
- Fallback: 'unknown' bucket for nil sectors; use `holding.sector` or `holding.data` if SecurityEnrichment missing
- Return `null` when user has no equities (not empty hash)
- JSON structure per Target Snapshot JSON Structure v1

### Non-Functional
- Only process equities
- Return null if no equity positions

## Acceptance Criteria
1. Sector weights sum to 1.0 (equities portion only)
2. Matches equity aggregates by sector
3. Returns `null` when no equities
4. Fallback handles missing enrichment data
5. Job runtime <5s

## Test Cases
```ruby
it "includes sector weights for equities" do
  create(:position, user: user, asset_class: 'equity', sector: 'technology', current_value: 280_000)
  create(:position, user: user, asset_class: 'equity', sector: 'healthcare', current_value: 150_000)
  create(:position, user: user, asset_class: 'cash', current_value: 100_000)

  FinancialSnapshotJob.perform_now(user)
  snapshot = user.financial_snapshots.last

  weights = snapshot.data['sector_weights']
  expect(weights['technology']).to be_within(0.01).of(0.651) # 280k/430k
  expect(weights['healthcare']).to be_within(0.01).of(0.349) # 150k/430k
  expect(weights.values.sum).to be_within(0.0001).of(1.0)
end

it "returns null for sector_weights when no equities" do
  create(:position, user: user, asset_class: 'cash', current_value: 100_000)

  FinancialSnapshotJob.perform_now(user)
  snapshot = user.financial_snapshots.last

  expect(snapshot.data['sector_weights']).to be_nil
end
```
