# PRD-2-01b: Reporting DataProvider Service Scaffold

## Overview
Implement minimal Reporting::DataProvider service to centralize aggregate queries for snapshots, promoting reusable patterns. Use ActiveRecord scopes and chainable relations for composability; defer Arel until dynamic multi-group or permission-based column selection is required. Start simple for v1 single-user, with hooks for future groups/sharing/security.

## Requirements

### Functional
- **Service**: `Reporting::DataProvider.new(user)` initialized with user context
- **Core Methods**:
  - `core_aggregates` - Returns hash with `total_net_worth`, `delta_day`, `delta_30d`
  - `asset_allocation_breakdown` - Returns hash with asset class percentages
  - `sector_weights` - Returns hash with equity sector percentages
  - `top_holdings(limit = 10)` - Returns array of top holdings hashes
  - `monthly_transaction_summary` - Returns hash with income/expenses/categories
  - `historical_trends(days = 30)` - Returns array of {date, value} hashes
  - `sync_freshness` - Returns hash with `{stale: boolean, last_sync_at: timestamp}`

- **Chainability**: Demonstrate with at least one example:
  - `with_date_range(start_date, end_date)` - Returns self after storing date filters
  - Call pattern: `provider.with_date_range(30.days.ago, Date.current).core_aggregates`

- **Composability**: Use ActiveRecord scopes and relations:
  - Primary approach: `user.positions.where(asset_class: 'equity').sum(:current_value)`
  - Use `.merge()` for combining scopes
  - Use `.or()` for alternative conditions
  - Chain methods that return relations

- **Security**: Basic user scoping
  - All queries scoped to `user` instance variable
  - Hooks for future Pundit/authorize! (comments only for v1)

- **Helper Methods**:
  - `normalize_percentages(hash)` - Forces sum to exactly 1.0 by adjusting largest value
  - `build_snapshot_hash` - Orchestrates all methods to build complete snapshot JSON

- **Export Stubs** (for future BI):
  - `to_json` - Returns snapshot hash as JSON string
  - `to_csv` - Returns basic CSV format (stub implementation)
  - `to_tableau_json` - Returns flattened hash for Tableau (stub)

### Non-Functional
- **Performant**: Memoize results using `@_memoized_*` pattern; <2s overhead for all aggregates
- **Composable**: Design for AI-coder extension (comment hooks for mix-ins like GroupFilter)
- **Idempotent**: Pure queries, no side effects, safe to call multiple times
- **No Arel for v1**: Explicitly avoid Arel unless hitting AR limitation

## Architectural Context
- **Rails**: Plain Ruby class in `app/services/reporting/data_provider.rb`
- **PostgreSQL**: Leverage existing indexes on Position, Account, Liability, Transaction models
- **Integration**: Used in FinancialSnapshotJob (PRD-2-02) for JSON generation; assumes synced models
- **Reference**: Follow Rails service patterns; no UI in this PRD

## Acceptance Criteria
1. `DataProvider.new(user).core_aggregates` returns expected hash (matches direct AR sums)
2. All methods return correct data types (hashes/arrays)
3. Handles no data gracefully (empty hashes/arrays, not errors)
4. Security: All queries scoped to user only (no cross-user data leakage)
5. Chainable: `provider.with_date_range(...).core_aggregates` works
6. Memoization: Second call to same method doesn't re-query database
7. Export stubs return formatted data (JSON string, CSV stub, Tableau stub)
8. Runtime <2s for full `build_snapshot_hash` with 500 positions + 1000 transactions
9. `normalize_percentages` ensures sum == 1.0 within 0.0001 tolerance
10. `sync_freshness` correctly identifies stale data (>36h old)

## Test Cases

### Service Spec (RSpec)

```ruby
require 'rails_helper'

RSpec.describe Reporting::DataProvider do
  let(:user) { create(:user) }
  let(:other_user) { create(:user) }
  let(:provider) { described_class.new(user) }

  before do
    # Seed test data
    create_list(:position, 5, user: user, asset_class: 'equity', current_value: 100_000)
    create_list(:position, 3, user: user, asset_class: 'cash', current_value: 50_000)
    create(:account, user: user, current_balance: 25_000)
    create(:liability, user: user, current_balance: 75_000)

    # Other user data (should not appear)
    create(:position, user: other_user, current_value: 999_999)
  end

  describe '#core_aggregates' do
    it 'computes total net worth correctly' do
      # 5 equity @ 100k + 3 cash @ 50k + 1 account @ 25k - 1 liability @ 75k
      # = 500k + 150k + 25k - 75k = 600k
      result = provider.core_aggregates

      expect(result[:total_net_worth]).to eq(600_000)
    end

    it 'returns zero for delta_day when no previous snapshot' do
      result = provider.core_aggregates
      expect(result[:delta_day]).to eq(0)
    end

    it 'computes delta_day from previous snapshot' do
      create(:financial_snapshot, user: user, snapshot_at: 1.day.ago, data: {
        total_net_worth: 590_000
      }, status: :complete)

      result = provider.core_aggregates
      expect(result[:delta_day]).to eq(10_000) # 600k - 590k
    end

    it 'computes delta_30d from 30-day snapshot' do
      create(:financial_snapshot, user: user, snapshot_at: 30.days.ago, data: {
        total_net_worth: 550_000
      }, status: :complete)

      result = provider.core_aggregates
      expect(result[:delta_30d]).to eq(50_000) # 600k - 550k
    end

    it 'uses nearest snapshot if exact date missing' do
      create(:financial_snapshot, user: user, snapshot_at: 2.days.ago, data: {
        total_net_worth: 595_000
      }, status: :complete)

      result = provider.core_aggregates
      expect(result[:delta_day]).to eq(5_000) # Falls back to nearest
    end

    it 'does not leak other user data' do
      result = provider.core_aggregates
      expect(result[:total_net_worth]).to eq(600_000) # Not 1.5M+
    end

    it 'handles negative net worth' do
      user.positions.destroy_all
      user.accounts.destroy_all
      create(:liability, user: user, current_balance: 500_000)

      result = provider.core_aggregates
      expect(result[:total_net_worth]).to eq(-500_000)
    end
  end

  describe '#asset_allocation_breakdown' do
    it 'returns percentages summing to 1.0' do
      result = provider.asset_allocation_breakdown

      expect(result).to be_a(Hash)
      expect(result.values.sum).to be_within(0.0001).of(1.0)
    end

    it 'computes correct percentages' do
      # 500k equity + 150k cash = 650k total positions
      result = provider.asset_allocation_breakdown

      expect(result[:equity]).to be_within(0.01).of(0.769) # 500/650
      expect(result[:cash]).to be_within(0.01).of(0.231)   # 150/650
    end

    it 'returns empty hash when no positions' do
      user.positions.destroy_all
      result = provider.asset_allocation_breakdown

      expect(result).to eq({})
    end

    it 'uses fallback for uncategorized' do
      create(:position, user: user, asset_class: nil, current_value: 10_000)
      result = provider.asset_allocation_breakdown

      expect(result).to have_key(:other)
    end
  end

  describe '#sector_weights' do
    before do
      # Update equity positions with sectors
      user.positions.where(asset_class: 'equity').each_with_index do |pos, i|
        sector = ['technology', 'healthcare', 'financials'][i % 3]
        pos.update!(sector: sector)
      end
    end

    it 'returns percentages for equity sectors only' do
      result = provider.sector_weights

      expect(result).to be_a(Hash)
      expect(result.keys).to all(be_a(Symbol))
      expect(result.values.sum).to be_within(0.0001).of(1.0)
    end

    it 'returns null when user has no equities' do
      user.positions.where(asset_class: 'equity').destroy_all
      result = provider.sector_weights

      expect(result).to be_nil
    end

    it 'includes unknown bucket for missing enrichment' do
      user.positions.first.update!(sector: nil)
      result = provider.sector_weights

      expect(result).to have_key(:unknown)
    end
  end

  describe '#top_holdings' do
    it 'returns top N holdings sorted by value descending' do
      result = provider.top_holdings(3)

      expect(result).to be_an(Array)
      expect(result.size).to eq(3)
      expect(result.first[:value]).to be >= result.last[:value]
    end

    it 'includes ticker, name, value, pct_portfolio' do
      result = provider.top_holdings(1)

      holding = result.first
      expect(holding).to have_key(:ticker)
      expect(holding).to have_key(:name)
      expect(holding).to have_key(:value)
      expect(holding).to have_key(:pct_portfolio)
    end

    it 'computes pct_portfolio correctly' do
      # Each equity position is 100k out of 650k total
      result = provider.top_holdings(1)

      expect(result.first[:pct_portfolio]).to be_within(0.01).of(0.154) # 100k/650k
    end

    it 'returns empty array when no positions' do
      user.positions.destroy_all
      result = provider.top_holdings

      expect(result).to eq([])
    end
  end

  describe '#monthly_transaction_summary' do
    before do
      create(:transaction, user: user, amount: 5000, category: 'Salary', date: 15.days.ago)
      create(:transaction, user: user, amount: 3000, category: 'Bonus', date: 10.days.ago)
      create(:transaction, user: user, amount: -1500, category: 'Rent', date: 20.days.ago)
      create(:transaction, user: user, amount: -500, category: 'Groceries', date: 5.days.ago)
      create(:transaction, user: user, amount: 1000, category: 'Old', date: 60.days.ago) # Outside 30d
    end

    it 'returns income and expenses for last 30 days' do
      result = provider.monthly_transaction_summary

      expect(result[:income]).to eq(8_000)   # 5k + 3k
      expect(result[:expenses]).to eq(2_000) # 1.5k + 0.5k
    end

    it 'returns top categories' do
      result = provider.monthly_transaction_summary

      expect(result[:top_categories]).to be_an(Array)
      expect(result[:top_categories].first).to have_key(:category)
      expect(result[:top_categories].first).to have_key(:amount)
    end

    it 'returns zeros when no transactions' do
      user.transactions.destroy_all
      result = provider.monthly_transaction_summary

      expect(result[:income]).to eq(0)
      expect(result[:expenses]).to eq(0)
      expect(result[:top_categories]).to eq([])
    end
  end

  describe '#historical_trends' do
    before do
      create(:financial_snapshot, user: user, snapshot_at: 5.days.ago,
             data: { total_net_worth: 580_000 }, status: :complete)
      create(:financial_snapshot, user: user, snapshot_at: 3.days.ago,
             data: { total_net_worth: 590_000 }, status: :complete)
      create(:financial_snapshot, user: user, snapshot_at: 1.day.ago,
             data: { total_net_worth: 595_000 }, status: :complete)
    end

    it 'returns array of date/value hashes sorted ascending' do
      result = provider.historical_trends(10)

      expect(result).to be_an(Array)
      expect(result.size).to eq(3)
      expect(result.first[:date]).to be < result.last[:date]
      expect(result.first[:value]).to eq(580_000)
    end

    it 'limits to specified days' do
      result = provider.historical_trends(4)

      expect(result.size).to eq(2) # Only 3-day and 1-day snapshots
    end

    it 'returns empty array when no history' do
      user.financial_snapshots.destroy_all
      result = provider.historical_trends

      expect(result).to eq([])
    end
  end

  describe '#sync_freshness' do
    it 'returns not stale when recent sync' do
      create(:item, user: user, last_successful_update: 1.hour.ago)
      result = provider.sync_freshness

      expect(result[:stale]).to be false
      expect(result[:last_sync_at]).to be_within(2.minutes).of(1.hour.ago)
    end

    it 'returns stale when sync >36h old' do
      create(:item, user: user, last_successful_update: 48.hours.ago)
      result = provider.sync_freshness

      expect(result[:stale]).to be true
    end

    it 'returns stale when no syncs exist' do
      result = provider.sync_freshness

      expect(result[:stale]).to be true
      expect(result[:last_sync_at]).to be_nil
    end
  end

  describe 'chainability' do
    describe '#with_date_range' do
      it 'returns self for chaining' do
        result = provider.with_date_range(30.days.ago, Date.current)
        expect(result).to eq(provider)
      end

      it 'filters data based on date range (demonstration)' do
        # Future: Could filter positions/transactions by date
        # For now, just demonstrate the pattern works
        result = provider.with_date_range(7.days.ago, Date.current).core_aggregates

        expect(result).to be_a(Hash)
        expect(result).to have_key(:total_net_worth)
      end
    end
  end

  describe 'memoization' do
    it 'does not re-query on second call' do
      # First call
      expect {
        provider.core_aggregates
      }.to make_database_queries(count: be > 0)

      # Second call should be memoized
      expect {
        provider.core_aggregates
      }.to make_database_queries(count: 0)
    end

    it 'memoizes each method independently' do
      provider.core_aggregates
      expect {
        provider.asset_allocation_breakdown
      }.to make_database_queries(count: be > 0)
    end
  end

  describe '#normalize_percentages' do
    it 'forces sum to exactly 1.0' do
      raw = { equity: 0.333, cash: 0.333, bonds: 0.333 }
      result = provider.send(:normalize_percentages, raw)

      expect(result.values.sum).to eq(1.0)
    end

    it 'adjusts largest category' do
      raw = { equity: 0.5, cash: 0.3, bonds: 0.19 } # Sum = 0.99
      result = provider.send(:normalize_percentages, raw)

      expect(result[:equity]).to eq(0.51) # Largest adjusted by +0.01
      expect(result.values.sum).to eq(1.0)
    end

    it 'handles rounding precision' do
      raw = { a: 0.1, b: 0.2, c: 0.3, d: 0.4 }
      result = provider.send(:normalize_percentages, raw)

      expect(result.values.sum).to be_within(0.0001).of(1.0)
    end
  end

  describe '#build_snapshot_hash' do
    it 'returns complete snapshot structure' do
      result = provider.build_snapshot_hash

      expect(result).to have_key(:schema_version)
      expect(result).to have_key(:as_of)
      expect(result).to have_key(:total_net_worth)
      expect(result).to have_key(:asset_allocation)
      expect(result).to have_key(:disclaimer)
    end

    it 'includes all required fields' do
      result = provider.build_snapshot_hash

      expect(result[:schema_version]).to eq(1)
      expect(result[:disclaimer]).to eq("Educational simulation only – not financial advice")
    end

    it 'completes within performance baseline' do
      expect {
        provider.build_snapshot_hash
      }.to perform_under(2).sec
    end
  end

  describe 'export stubs' do
    describe '#to_json' do
      it 'returns JSON string' do
        result = provider.to_json

        expect(result).to be_a(String)
        parsed = JSON.parse(result)
        expect(parsed).to have_key('total_net_worth')
      end
    end

    describe '#to_csv' do
      it 'returns CSV stub' do
        result = provider.to_csv

        expect(result).to be_a(String)
        expect(result).to include('total_net_worth')
      end
    end

    describe '#to_tableau_json' do
      it 'returns flattened hash' do
        result = provider.to_tableau_json

        expect(result).to be_a(Hash)
        # Flattened: no nested hashes
        expect(result.values).to all(satisfy { |v| !v.is_a?(Hash) || v.empty? })
      end
    end
  end

  describe 'edge cases' do
    it 'handles user with no data' do
      user.positions.destroy_all
      user.accounts.destroy_all
      user.liabilities.destroy_all
      user.transactions.destroy_all

      result = provider.build_snapshot_hash

      expect(result[:total_net_worth]).to eq(0)
      expect(result[:asset_allocation]).to eq({})
      expect(result[:sector_weights]).to be_nil
    end

    it 'handles very large portfolios' do
      user.positions.destroy_all
      create(:position, user: user, current_value: 1_500_000_000)

      result = provider.core_aggregates

      expect(result[:total_net_worth]).to eq(1_500_000_000)
    end
  end
end
```

```ruby
require 'benchmark'

RSpec.describe 'DataProvider Performance', type: :benchmark do
  let(:user) { create(:user) }
  let(:provider) { Reporting::DataProvider.new(user) }

  before do
    # Seed realistic data volume
    create_list(:position, 500, user: user)
    create_list(:transaction, 1000, user: user, date: (1..30).to_a.sample.days.ago)
    create_list(:financial_snapshot, 30, user: user) do |snap, i|
      snap.snapshot_at = i.days.ago
      snap.data = { total_net_worth: 600_000 + rand(10_000) }
    end
  end

  it 'completes full snapshot build under 2 seconds' do
    time = Benchmark.realtime do
      provider.build_snapshot_hash
    end

    expect(time).to be < 2.0
    puts "Full snapshot build: #{time.round(3)}s"
  end

  it 'benefits from memoization' do
    first_time = Benchmark.realtime { provider.core_aggregates }
    second_time = Benchmark.realtime { provider.core_aggregates }

    expect(second_time).to be < (first_time * 0.1) # 90% faster
  end
end
```
