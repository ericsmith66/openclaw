# PRD-2-02: Daily FinancialSnapshotJob – Core Aggregates

## Overview
Implement FinancialSnapshotJob to run daily, computing core net worth aggregates (total, deltas) via DataProvider and storing as JSON in FinancialSnapshot.data. This powers Epic 3 net worth hero card and changes display.

## Status
**Branch**: `feature/epic-2-financial-snapshots`
**Dependencies**: PRD-2-01 (Model), PRD-2-01b (DataProvider) must be complete

## Requirements

### Functional
- **Job**: `FinancialSnapshotJob.perform_later(user)`
  - Uses DataProvider to compute core aggregates
  - Stores result in `FinancialSnapshot.data` with status `:complete`
  - Adds `disclaimer` field to data
  - Checks sync freshness and sets `:stale` status if >36h

- **Data Computed** (via DataProvider):
  - `total_net_worth`: Sum of Position.current_value + Account.balances - Liability.balances
  - `delta_day`: Difference vs previous snapshot (or nearest if exact missing, preserve sign)
  - `delta_30d`: Difference vs 30 days ago snapshot
  - `as_of`: Current date in CST

- **Schedule**: Sidekiq cron daily at midnight CST (config/sidekiq.yml)
  - **V1 Note**: Simple midnight cron acceptable for 10-50 users
  - **Future**: Add staggering via `user.id % 60` or batch job when user count grows

- **Error Handling**:
  - `rescue_from StandardError` → update status: `:error`, store error message in data
  - `discard_on ActiveRecord::RecordNotUnique` → don't retry duplicates
  - `retry_on` with exponential backoff, max 3 attempts
  - Log errors for monitoring

- **Empty Data Handling**:
  - If `total_net_worth = 0` due to no positions/accounts, set status `:empty`
  - Include warning in data quality

- **Idempotent**: Safe to rerun on existing date
  - Find or initialize snapshot for user + today
  - Update if status `:error` or `:pending`
  - Skip if status `:complete` (unless forced)

### Non-Functional
- **Performance**: <5s per user with 500 positions + 1000 transactions
- **Idempotent**: Safe to rerun without side effects
- **Logging**: Error if sync stale, set `:stale` status
- **Assumptions**: Position/Account/Liability models have valid data from prior syncs
  - Fail explicitly if sums return nil/zero unexpectedly

## Architectural Context
- **Rails**: Sidekiq job using ActiveJob
- **Service**: Uses `Reporting::DataProvider` for all aggregations
- **PostgreSQL**: JSONb for data storage
- **Integration**: Assumes synced Plaid data; no Plaid API calls in this job
- **Reference**: No UI; follows simplicity principles from style guide

## Acceptance Criteria
1. Job creates snapshot with correct `total_net_worth` (matches model sums via DataProvider)
2. `delta_day` accurate vs previous (or nearest available)
3. `delta_30d` = 0 if no 30-day prior; correct diff otherwise
4. Cron entry in `config/sidekiq.yml` schedules daily at midnight CST
5. Job logs warning and sets `:stale` if no recent sync (<36h)
6. Rerun on existing date updates JSON if status `:error` or `:pending`
7. Retry handles transient errors (e.g., DB timeout)
8. Empty data sets status `:empty` with warning
9. Disclaimer field included in JSON
10. Performance: <5s execution time
11. Negative net worth handled correctly
12. Concurrent job runs don't create duplicates (unique constraint)

require 'rails_helper'

RSpec.describe FinancialSnapshotJob, type: :job do
  include ActiveJob::TestHelper

  let(:user) { create(:user) }

  before do
    # Seed typical user data
    create_list(:position, 10, user: user, current_value: 50_000, asset_class: 'equity')
    create(:account, user: user, current_balance: 100_000)
    create(:liability, user: user, current_balance: 50_000)

    # Mock recent sync
    create(:item, user: user, last_successful_update: 1.hour.ago)
  end

  describe '#perform' do
    it 'creates snapshot with correct core aggregates' do
      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot).to be_present
      expect(snapshot.status).to eq('complete')

      # 10 positions @ 50k + 100k account - 50k liability = 550k
      expect(snapshot.data['total_net_worth']).to eq(550_000)
      expect(snapshot.data['as_of']).to eq(Date.current.to_s)
      expect(snapshot.data['disclaimer']).to eq("Educational simulation only – not financial advice")
    end

    it 'computes delta_day from previous snapshot' do
      create(:financial_snapshot, user: user, snapshot_at: 1.day.ago,
             data: { total_net_worth: 540_000 }, status: :complete)

      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot.data['delta_day']).to eq(10_000) # 550k - 540k
    end

    it 'computes delta_30d from 30-day snapshot' do
      create(:financial_snapshot, user: user, snapshot_at: 30.days.ago,
             data: { total_net_worth: 500_000 }, status: :complete)

      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot.data['delta_30d']).to eq(50_000) # 550k - 500k
    end

    it 'uses nearest snapshot when exact date missing' do
      create(:financial_snapshot, user: user, snapshot_at: 2.days.ago,
             data: { total_net_worth: 545_000 }, status: :complete)

      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot.data['delta_day']).to eq(5_000) # Falls back to nearest
    end

    it 'sets deltas to 0 when no previous snapshots' do
      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot.data['delta_day']).to eq(0)
      expect(snapshot.data['delta_30d']).to eq(0)
    end

    it 'preserves delta sign correctly' do
      create(:financial_snapshot, user: user, snapshot_at: 1.day.ago,
             data: { total_net_worth: 560_000 }, status: :complete)

      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot.data['delta_day']).to eq(-10_000) # 550k - 560k = -10k
    end
  end

  describe 'error handling' do
    it 'sets status to error and stores message on failure' do
      allow_any_instance_of(Reporting::DataProvider).to receive(:core_aggregates)
        .and_raise(StandardError, 'Test error')

      perform_enqueued_jobs do
        expect {
          described_class.perform_later(user)
        }.to raise_error(StandardError)
      end

      snapshot = user.financial_snapshots.last
      expect(snapshot.status).to eq('error')
      expect(snapshot.data['error']).to include('Test error')
    end

    it 'retries on transient errors with exponential backoff' do
      call_count = 0
      allow_any_instance_of(Reporting::DataProvider).to receive(:build_snapshot_hash) do
        call_count += 1
        raise StandardError, 'Transient error' if call_count < 3
        { total_net_worth: 550_000 }
      end

      perform_enqueued_jobs { described_class.perform_later(user) }

      expect(call_count).to eq(3)
      snapshot = user.financial_snapshots.last
      expect(snapshot.status).to eq('complete')
    end

    it 'discards on duplicate record error' do
      create(:financial_snapshot, user: user, snapshot_at: Date.current, status: :complete)

      # Try to create duplicate
      expect {
        perform_enqueued_jobs { described_class.perform_later(user) }
      }.not_to change { user.financial_snapshots.count }
    end
  end

  describe 'stale sync detection' do
    it 'sets status to stale when sync >36h old' do
      user.items.update_all(last_successful_update: 48.hours.ago)

      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot.status).to eq('stale')
      expect(snapshot.data['data_quality']['warnings']).to include(match(/stale/i))
    end

    it 'sets status to complete when sync is recent' do
      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot.status).to eq('complete')
    end
  end

  describe 'empty data handling' do
    it 'sets status to empty when user has no financial data' do
      user.positions.destroy_all
      user.accounts.destroy_all
      user.liabilities.destroy_all

      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot.status).to eq('empty')
      expect(snapshot.data['total_net_worth']).to eq(0)
      expect(snapshot.data['data_quality']['warnings']).to include(match(/no.*data/i))
    end
  end

  describe 'idempotence' do
    it 'updates existing pending snapshot instead of creating new' do
      existing = create(:financial_snapshot, user: user, snapshot_at: Date.current,
                       status: :pending, data: {})

      perform_enqueued_jobs { described_class.perform_later(user) }

      expect(user.financial_snapshots.count).to eq(1)
      expect(existing.reload.status).to eq('complete')
      expect(existing.data['total_net_worth']).to eq(550_000)
    end

    it 'updates existing error snapshot' do
      existing = create(:financial_snapshot, user: user, snapshot_at: Date.current,
                       status: :error, data: { error: 'Previous error' })

      perform_enqueued_jobs { described_class.perform_later(user) }

      expect(user.financial_snapshots.count).to eq(1)
      expect(existing.reload.status).to eq('complete')
    end

    it 'skips existing complete snapshot unless forced' do
      existing = create(:financial_snapshot, user: user, snapshot_at: Date.current,
                       status: :complete, data: { total_net_worth: 999_999 })

      perform_enqueued_jobs { described_class.perform_later(user) }

      # Should not update complete snapshot
      expect(existing.reload.data['total_net_worth']).to eq(999_999)
    end
  end

  describe 'edge cases' do
    it 'handles negative net worth correctly' do
      user.positions.destroy_all
      user.accounts.destroy_all
      create(:liability, user: user, current_balance: 100_000)

      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot.data['total_net_worth']).to eq(-100_000)
      expect(snapshot.status).to eq('complete')
    end

    it 'handles very large portfolios' do
      user.positions.destroy_all
      create(:position, user: user, current_value: 1_500_000_000)

      perform_enqueued_jobs { described_class.perform_later(user) }

      snapshot = user.financial_snapshots.last
      expect(snapshot.data['total_net_worth']).to eq(1_500_000_000)
    end

    it 'prevents concurrent duplicate creation with unique constraint' do
      threads = 2.times.map do
        Thread.new { described_class.perform_now(user) rescue nil }
      end
      threads.each(&:join)

      expect(user.financial_snapshots.where(snapshot_at: Date.current).count).to eq(1)
    end
  end

  describe 'performance' do
    it 'completes within 5 seconds for 500 positions' do
      user.positions.destroy_all
      create_list(:position, 500, user: user, current_value: 1000)
      create_list(:transaction, 1000, user: user)

      time = Benchmark.realtime do
        perform_enqueued_jobs { described_class.perform_later(user) }
      end

      expect(time).to be < 5.0
    end
  end
end

### Implementation Steps

1. Generate job: `rails g job financial_snapshot`

2. Implement job class:
   ```ruby
   class FinancialSnapshotJob < ApplicationJob
     queue_as :default

     retry_on StandardError, wait: :exponentially_longer, attempts: 3
     discard_on ActiveRecord::RecordNotUnique

     def perform(user, force: false)
       snapshot = find_or_initialize_snapshot(user)

       # Skip if already complete (unless forced)
       return if snapshot.complete? && !force

       # Build data via DataProvider
       provider = Reporting::DataProvider.new(user)
       snapshot_data = provider.build_snapshot_hash

       # Check sync freshness
       sync_meta = provider.sync_freshness
       status = determine_status(snapshot_data, sync_meta)

       # Save snapshot
       snapshot.update!(
         data: snapshot_data,
         status: status,
         schema_version: 1
       )

     rescue StandardError => e
       Rails.logger.error("FinancialSnapshotJob failed for user #{user.id}: #{e.message}")
       snapshot.update!(
         status: :error,
         data: { error: e.message, schema_version: 1 }
       )
       raise # Still raise to trigger retry logic
     end

     private

     def find_or_initialize_snapshot(user)
       user.financial_snapshots.find_or_initialize_by(
         snapshot_at: Date.current
       )
     end

     def determine_status(data, sync_meta)
       return :empty if data[:total_net_worth] == 0
       return :stale if sync_meta[:stale]
       :complete
     end
   end
   ```

3. Add to `config/sidekiq.yml`:
   ```yaml
   :cron:
     financial_snapshot:
       cron: "0 0 * * *" # Daily at midnight CST
       class: FinancialSnapshotJob
       # Future: Add staggering when user count grows
       # cron: "*/5 * * * *" # Every 5 min, job internally staggers
   ```

4. Update `Reporting::DataProvider#build_snapshot_hash` to include disclaimer

5. Write specs

6. Run tests: `rspec spec/jobs/financial_snapshot_job_spec.rb`
