## Epic Goal

Stand up Legion as a working Rails 8 project with the `agent_desk` gem integrated, the ROR agent team configured, and a curated knowledge base. At the end of Epic 0, an agent can be dispatched through the gem's Runner, communicate with SmartProxy, and return a response ‚Äî all verified by passing tests.

**This is not about building features.** This is about having a clean, tested foundation that Epic 1 can build on without any baggage.

## Key Decisions Locked In

**Architecture / Boundaries:**
- CLI-first ‚Äî no UI until Epic 4; Workspace component copied but not rendered
- Gem's MessageBus is in-memory (`CallbackBus`), MemoryStore is file-based (JSON) ‚Äî no PostgreSQL models needed for gem operation
- `agent_desk` gem lives in-repo at `gems/agent_desk/` (not an external dependency)
- SmartProxy is external always-on infrastructure at `localhost:3002` ‚Äî not managed by Legion

**Naming / Standards:**
- All "Junie" naming refactored to "Agent" naming throughout knowledge base (guidelines, task logs, templates, RULES.md)
- `knowledge_base/prds-junie-log/` ‚Üí `knowledge_base/task-logs/`
- `projects/` sub-project structure retained in agent-guidelines for future non-Legion work

**Secrets / Config:**
- `.env` holds secrets (SmartProxy URL, token) ‚Äî already in `.gitignore`
- `.env.example` provides placeholder documentation ‚Äî committed to repo
- Gem `bin/` scripts (`agent_desk_cli`, `smoke_test`, `model_compatibility_test`) use ENV vars for SmartProxy connection

**Testing / Quality:**
- `scripts/pre-qa-validate.sh` carried forward from agent-forge ‚Äî adapted for Legion paths
- `bin/smoke_test` (10-step pipeline) and `bin/agent_desk_cli` are end-to-end success criteria
- Minitest only ‚Äî never RSpec

**Phase Skips (Bootstrap Exception):**
- Œ¶4 skipped ‚Äî PRD details already substantial in consolidated epic doc
- Œ¶5/Œ¶6 skipped ‚Äî Architect agent not yet configured (chicken-and-egg for bootstrap)
